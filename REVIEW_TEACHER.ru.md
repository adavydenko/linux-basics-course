# Отчёт ревизии: преподаватель курса

Перспектива: преподаватель технического вуза, которому предложили вести курс «Основы Linux» (2–3 курс прикладной математики, 1 семестр, 16 пар лекции + 16 пар практики). Пособие — основной учебник.

## Перспектива A — модуль за модулем

**00. Lab Environment.** Чётко описаны Docker-контейнер и VM, перечислены задания, которые требуют VM. Минусы: предполагается готовый `Dockerfile` в `lab-environment/`, но его наличие/качество не проверено мной — для семестра это критическая точка. Нет инструкции преподавателю «как раздать студентам единый образ».

**01. Введение.** Двухстраничный нарратив с «Бобом». Целей дисциплины (компетенции, итоговая аттестация, шкала) — нет. Для университета нужно поверх дописать рабочую программу.

**02. Shell Part I.** Базовые команды, навигация, `pwd/cd/ls/cp/mv/rm`, справка, Bash (alias, PATH, PS1). Стиль хорош. Критично: ни слова о пайплайнах/перенаправлении до конца модуля 02 — а в лабе уже используется `grep | cut`. Нет упоминания glob-шаблонов (`*`, `?`). `pushd/popd` дан без оговорки, что это украшение, путает новичков.

**03. Linux Core Concepts.** Ядро, оборудование, загрузка, runlevels, FHS. Описание ядра как «монолитное и модульное одновременно» — упрощение приемлемое. Таблица major-номеров (1=RAM, 3=HDD, 6=printer, 8=SCSI) устарела и местами неверна (на современных ядрах виртуальные диски — `253`, NVMe — `259`). Версионирование ядра `4.15.0-72-generic` — Linux уже на 6.x, надо обновить пример. Runlevels — для systemd-мира историческая справка, стоит так и обозначить.

**04. Package Management.** Параллель `dpkg/apt` vs `rpm/yum/dnf`. Корректно, но: нет ни слова про PPA, snap/flatpak, GPG-подписи репозиториев, `apt-mark hold`, `dpkg --configure -a`. Для университета — слишком сжато. Лаб по RPM в Ubuntu-окружении выполнить нельзя — нужна оговорка про второй контейнер.

**05. Shell Part II.** Сжатие, поиск (`find`/`locate`/`grep`), I/O-перенаправление, `vi`. По `find` дано только `-name` — для семестра это очень мало: ни `-type`, `-mtime`, `-size`, `-exec`, `-print0`. По `grep` нет ни слова про регулярные выражения и `-E`. Перенаправление: `2>&1` и `&>` не упомянуты вообще — это пробел. Process substitution `<(...)` и здесь, и далее не появляется.

**06. Security & File Permissions.** Учётки, `/etc/passwd`, `chmod`/`chown`, SSH/SCP, iptables, cron. Хорошие места: SSH-ключи объяснены толково. Слабые места: SUID/SGID/sticky bit пропущены полностью — это базовая программа по безопасности, без них объяснение `/tmp` и `passwd` невозможно. ACL (`getfacl/setfacl`), umask — нет. PAM, `/etc/shadow` упомянуты, но не разобраны. Cron не различает пользовательский и системный (`/etc/cron.d`, `/etc/crontab`). `iptables` — отдельный модуль, но в современном Ubuntu по умолчанию `nftables`/`ufw`; это противоречит модулю 17.

**07. Networking.** DNS, switching/routing, troubleshooting. `route` показан как живой инструмент — устаревший. `ip addr add` — без `nmcli`/`netplan`, то есть студент не научится реально настраивать постоянную сеть в Ubuntu 22.04. Нет ничего про TCP/UDP-handshake, MTU, NAT, VLAN.

**08. Storage in Linux.** Диски, разделы, ext4, LVM, плюс «история DAS/NAS/SAN». GPT/MBR упомянуты, но `parted`/`sgdisk` — нет, только `gdisk`. Про XFS, Btrfs, ZFS — одна строка. Нет swap (`mkswap`, `swapon`), `fsck`, `tune2fs`. LVM-раздел разумный, но snapshot'ы LVM — главная фишка — не упомянуты.

**09. Service management — systemd.** Хорошо: unit-файл с тремя секциями, `systemctl`, `journalctl`. Не хватает: таймеров systemd (`.timer` units — прямая альтернатива cron), `systemd-analyze`, target'ов как графа зависимостей (только `WantedBy`, нет `Wants`/`Requires`/`After`).

**10. Client Demonstration.** Сторителлинг-модуль. Для университета — балласт; либо превратить в задание-кейс, либо вынести во внеаудиторное чтение.

**11. Text Processing.** `head`/`tail`/`wc`/`cut`/`sort`/`uniq`/`grep`/`awk`/`sed`/`xargs`. Хороший модуль. Но `awk` дан на двух строках — для университета прикладной математики этого мало (это же фактически язык). Регулярные выражения нигде систематически не вводятся — а они нужны и здесь, и в `grep`, и в `sed`. Это самый явный пробел курса.

**12. Shell Scripting.** Переменные, `if`, `for`, `while`, `$?`, `set -euo pipefail`. Концептуально верно, но очень коротко: `case`, функции, массивы, `trap`, подстановка команд `$(...)`, арифметика `((...))`, проверки `[[ ... ]]` vs `[ ... ]`, here-doc и here-string — не упомянуты. На семестровом курсе скриптинг должен занять минимум 2 пары — материала тут на одну.

**13. Processes and Resources.** `ps`/`top`/`htop`, `jobs`/`fg`/`bg`, `kill`, `nice`/`renice`, `nohup`/`tmux`, `free`/`df`/`du`. Хороший практический минимум. Нет: сигналов как темы (только `kill -9`, без `SIGTERM`/`SIGHUP`/`SIGINT`), `pgrep`/`pkill`, `lsof`, `strace`, состояния процесса (R/S/D/Z), родитель/потомок, PPID, демонизация.

**14. Remote Work.** SSH-ключи, `~/.ssh/config`, `scp`/`rsync`, `tmux`. Дублирует модуль 06 в части SSH. `ssh -J` (jump host), `ProxyCommand`, port forwarding — отсутствуют.

**15. Git from Command Line.** Клон, status, add, commit, push/pull, `.gitignore`. Очень узко: нет веток, merge, rebase, конфликтов, `git log`, `git reflog`. Уместнее назвать «Git: минимальный цикл для сдачи лабораторных».

**16. Python from Command Line.** `venv`, `pip`, `requirements.txt`, перенаправления, `nohup`. Корректно для своей цели.

**17. Modern Networking.** `ip`, `ss`, `curl`, обзор `iptables`/`nftables`/`ufw`/`firewalld`. Логичное дополнение к 07; иронично, что в 06 уже подробно дан устаревший `iptables`. Порядок 06 → 07 → 17 надо перебирать.

**18. Linux Containers.** Docker/Podman, образ vs контейнер, bind mount, проброс портов, логи, exec. Хороший обзорный модуль. Не упомянуты: Dockerfile, `docker compose`, registry/тэги.

**19. Diagnostics.** Алгоритм диагностики. Самый зрелый модуль методически — но без `strace`/`lsof`/`tcpdump`/`iostat`/`vmstat` диагностика остаётся «детской».

**20. Final Project.** Каркас итогового задания. Чего нет: рубрик с баллами, чек-листа защиты, примера сданного проекта.

## Перспектива B — могу ли оценить студента

**Контрольные вопросы.** 95% — репродуктивные («что делает X», «чем X отличается от Y»). Аналитических («почему именно так устроено», «когда вы НЕ будете использовать X», «спроектируйте решение») — единицы. Студент, который зазубрит ответы, экзамен сдаст, но ничего реального не решит.

**Практические задания.** В большинстве модулей — пошаговые «сделайте 1, потом 2». Это лабораторные на минимум, но не задачи в академическом смысле. Нет: задач с открытым ответом, задач на проектирование, задач на отладку (вот сломанный конфиг, найдите ошибку), задач с подвохом. На семестр в текущем виде едва набирается 30-40% того, что нужно для нормальной оценки.

**Слепые зоны** (студент формально ответит, но реально применять не сможет):
- регулярные выражения (нигде не введены, но используются в `grep`/`sed`/`awk`);
- сигналы и реакция процессов на них (только `kill -9`);
- настройка постоянной сети в современном Ubuntu (Netplan/NetworkManager);
- отладка скриптов и системных проблем без `strace`/`lsof`/`tcpdump`;
- права в Unix полностью (SUID/SGID/sticky/umask/ACL);
- systemd-таймеры, dependency-граф юнитов;
- Dockerfile и компоновка контейнерных приложений;
- регулярная работа в Git с ветками, конфликтами и code review.

## Итоговый вердикт

**Статус сегодня — вспомогательное пособие, не основной учебник.** Текст хорошо написан (стиль выдержан, переходы плавные). Уверенная учебная подкладка — но без существенной доработки на семестр университетского курса ПМ не хватает: примерно 60% нужного содержания и 30% нужной задачной базы. Для самообучения и первых 4-6 пар — годится. Для роли единственного учебника на 16+16 — нет.

## Главные доработки по приоритету

1. **Банк задач.** Минимум 8-10 задач на модуль, включая 2-3 аналитических. Без этого нельзя ни ДЗ собрать, ни экзамен принять. Самая критичная брешь.

2. **Отдельный модуль «Регулярные выражения и обработка текста»** перед текущим модулем 11 либо вставкой в модуль 05. Без регэкспов половина материала по `grep`/`sed`/`awk`/`journalctl` повисает.

3. **Достройка пропусков по правам и процессам:** SUID/SGID/sticky, umask, ACL, сигналы как тема, `lsof`/`strace`/`pgrep`.

4. **Скриптинг расширить до двух полноценных модулей:** функции, массивы, `case`, `trap`, here-doc, отладка `set -x`, тестирование скриптов.

5. **Современная сеть и сервисы:** настройка сети через Netplan, systemd-таймеры как тема, Dockerfile хотя бы минимальный. Убрать `route`/`ifconfig` из основного текста в исторические врезки. Решить, что делать с `iptables` vs `nftables`/`ufw`.

6. **Методическая обвязка:** глоссарий терминов, ключи к практическим заданиям, чек-лист защиты итогового проекта, шкала баллов, формальные определения там, где сейчас живут аналогии.

7. **Реструктурировать порядок модулей:** объединить SSH из 06 и 14; перенести `iptables` в модуль 17 (или в приложение); 10-й сюжетный модуль убрать в опциональное чтение; модуль 19 (диагностика) разнести как сквозную линию по всем модулям.
