# Работа с текстовыми данными

В Linux большая часть администрирования и анализа начинается с текста: логи, CSV-файлы, списки пользователей, вывод команд. Умение быстро обработать текст в терминале экономит время и помогает проверять гипотезы без тяжёлых инструментов.

## Цели раздела

После раздела студент должен уметь:

- просматривать начало и конец файла;
- считать строки, слова и байты;
- выбирать колонки из текстовых данных;
- сортировать и удалять дубликаты;
- строить простые пайплайны обработки;
- понимать, когда достаточно shell-команд, а когда лучше писать полноценную программу.

## Базовые команды

Посмотреть первые строки файла:

```shell
$ head data.csv
$ head -n 20 data.csv
```

Посмотреть последние строки файла:

```shell
$ tail data.csv
$ tail -n 50 app.log
```

Следить за логом в реальном времени:

```shell
$ tail -f app.log
```

Посчитать строки, слова и байты:

```shell
$ wc data.csv
$ wc -l data.csv
```

## Выбор колонок: `cut`

`cut` удобен для простых таблиц с разделителем.

```shell
$ cut -d "," -f1 data.csv
$ cut -d ":" -f1 /etc/passwd
```

Здесь `-d` задаёт разделитель, а `-f` выбирает номер поля.

## Сортировка и уникальные значения

```shell
$ sort names.txt
$ sort names.txt | uniq
$ sort names.txt | uniq -c
$ sort names.txt | uniq -c | sort -nr
```

Комбинация `sort | uniq -c | sort -nr` часто используется для подсчёта частот.

## Поиск по шаблону

`grep` ищет строки, соответствующие шаблону:

```shell
$ grep "ERROR" app.log
$ grep -i "error" app.log
$ grep -v "DEBUG" app.log
$ grep -R "TODO" .
```

Полезные опции:

- `-i` — не учитывать регистр;
- `-v` — вывести строки, которые не совпали;
- `-R` — искать рекурсивно;
- `-n` — показать номера строк.

## `awk`

`awk` удобен, когда нужно работать с колонками и простыми вычислениями.

```shell
$ awk '{ print $1 }' access.log
$ awk -F "," '{ print $1, $3 }' data.csv
$ awk -F "," '{ sum += $3 } END { print sum }' data.csv
```

В `awk` переменная `$1` означает первое поле, `$2` — второе поле и так далее. `$0` означает всю строку.

## `sed`

`sed` применяет простые преобразования к потоку текста.

Заменить первое совпадение в строке:

```shell
$ sed 's/error/ERROR/' app.log
```

Заменить все совпадения:

```shell
$ sed 's/error/ERROR/g' app.log
```

Удалить пустые строки:

```shell
$ sed '/^$/d' notes.txt
```

## `xargs`

`xargs` превращает входной поток в аргументы команды.

```shell
$ cat files.txt | xargs ls -l
$ find . -name "*.log" | xargs wc -l
```

Если имена файлов могут содержать пробелы, безопаснее использовать `find -print0` и `xargs -0`:

```shell
$ find . -name "*.log" -print0 | xargs -0 wc -l
```
