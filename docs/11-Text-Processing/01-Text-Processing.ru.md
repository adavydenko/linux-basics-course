# Работа с текстовыми данными

В предыдущих модулях вы уже видели, как Linux отдаёт результат: длинные журналы systemd, списки процессов из `ps aux`, дампы конфигурации, файлы вроде `/etc/passwd`. Эти данные — почти всегда текст, и почти всегда его слишком много, чтобы читать глазами целиком. Чтобы извлечь из текста ответ на конкретный вопрос — «сколько ошибок за сутки», «какие пять IP стучат чаще всех», «есть ли в логе упоминание этого пользователя» — нужен отдельный набор инструментов. Этот модуль собирает базу: `head`, `tail`, `wc`, `cut`, `sort`, `uniq`, `grep`, `awk`, `sed` и связку из них через конвейеры.

> **Пререквизит.** Прежде чем браться за этот модуль, прочитайте [приложение «Регулярные выражения»](../A6-Appendix-Regex/01-Regular-Expressions.ru.md): дальше будут конвейеры с `grep -E`, `sed -E`, `awk`, и без знания регэкспов половина возможностей этих команд останется за кадром.

## Цели раздела

После раздела студент должен уметь:

- просматривать начало и конец файла;
- считать строки, слова и байты;
- выбирать колонки из текстовых данных;
- сортировать и удалять дубликаты;
- строить простые пайплайны обработки;
- понимать, когда достаточно shell-команд, а когда лучше писать полноценную программу.

## Базовые команды

Прежде чем обрабатывать файл, полезно просто на него взглянуть — но открывать целиком лог на сто тысяч строк незачем. Команда `head` показывает только начало файла, по умолчанию первые десять строк, а флаг `-n` задаёт, сколько именно строк вам нужно:

```shell
$ head data.csv
$ head -n 20 data.csv
```

Зеркальная ей `tail` (хвост) показывает конец файла. Для логов это обычно интереснее начала: свежие записи всегда внизу.

```shell
$ tail data.csv
$ tail -n 50 app.log
```

У `tail` есть особенно ценный режим — флаг `-f` (follow). Команда не завершается, а остаётся «прицепленной» к файлу и дописывает на экран каждую новую строку. Так удобно наблюдать за логом работающего приложения в реальном времени; выйти из режима — `Ctrl+C`:

```shell
$ tail -f app.log
```

Чтобы оценить объём данных, не читая их, есть `wc` (word count). Без флагов она печатает три числа — количество строк, слов и байтов; с флагом `-l` (lines) оставляет только строки, и это самый частый вариант — «сколько вообще записей в файле»:

```shell
$ wc data.csv
$ wc -l data.csv
```

## Выбор колонок: `cut`

Текстовые данные часто разбиты на колонки — CSV-таблицы, системные файлы вроде `/etc/passwd`. Когда из такой строки нужно вытащить одну колонку, на помощь приходит `cut`. Идея простая: вы говорите команде, каким символом разделены поля и какое из них вам нужно.

```shell
$ cut -d "," -f1 data.csv
$ cut -d ":" -f1 /etc/passwd
```

Флаг `-d` (delimiter) задаёт символ-разделитель, а `-f` (field) — номер нужного поля. В первом примере мы берём первую колонку CSV, во втором — имена пользователей из `/etc/passwd`, где поля разделены двоеточием.

## Сортировка и уникальные значения

Часто нужно не просто посмотреть данные, а понять, что в них повторяется. Здесь работают две команды в связке. `sort` упорядочивает строки, а `uniq` схлопывает идущие подряд одинаковые строки в одну — поэтому `uniq` почти всегда идёт после `sort`: без сортировки одинаковые строки могут оказаться в разных местах файла, и `uniq` их не заметит.

```shell
$ sort names.txt
$ sort names.txt | uniq
$ sort names.txt | uniq -c
$ sort names.txt | uniq -c | sort -nr
```

Флаг `-c` (count) у `uniq` добавляет к каждой строке счётчик повторений, а финальный `sort -nr` сортирует результат по этому числу (`-n` — как числа, а не как текст; `-r` — по убыванию). Связку `sort | uniq -c | sort -nr` стоит запомнить целиком: это стандартный способ построить «топ» — самые частые ошибки в логе, самые активные IP-адреса и так далее.

## Поиск по шаблону

Когда файл большой, а нужна лишь часть строк, на сцену выходит `grep` — команда, которая оставляет только строки, подходящие под заданный шаблон:

```shell
$ grep "ERROR" app.log
$ grep -i "error" app.log
$ grep -v "DEBUG" app.log
$ grep -R "TODO" .
```

Несколько флагов меняют поведение `grep` и пригодятся постоянно:

- `-i` (ignore case) — искать без учёта регистра, так `error` найдёт и `ERROR`, и `Error`;
- `-v` (invert) — переворачивает логику: выводит строки, которые НЕ совпали. Удобно, чтобы выкинуть из лога весь шум вроде `DEBUG`;
- `-R` (recursive) — искать не в одном файле, а во всех файлах каталога и его подкаталогов;
- `-n` (number) — показывать номера строк, чтобы потом легко найти нужное место в файле.

## `awk`

`cut` умеет вырезать колонки, но не умеет считать. Когда нужно не только выбрать поля, но и что-то с ними сделать — сложить, отфильтровать, переформатировать — берут `awk`. Это маленький язык обработки текста, и для начала достаточно знать его модель: `awk` читает файл строку за строкой и для каждой выполняет программу в фигурных скобках.

```shell
$ awk '{ print $1 }' access.log
$ awk -F "," '{ print $1, $3 }' data.csv
$ awk -F "," '{ sum += $3 } END { print sum }' data.csv
```

Внутри программы `awk` автоматически разбивает строку на поля: `$1` — первое поле, `$2` — второе и так далее, а `$0` — вся строка целиком. Флаг `-F` (field separator) задаёт разделитель полей, как `-d` у `cut`. Третий пример показывает, на что `awk` способен сверх простого вырезания: он накапливает сумму третьей колонки в переменной `sum`, а блок `END` выполняется один раз после всего файла и печатает итог.

> **Типичные ошибки.**
>
> - **Поля без `-F` для CSV.** `awk '{ print $1 }' data.csv` для CSV-файла вернёт всю строку до первого пробела, а не первое поле — по умолчанию `awk` разделяет поля пробелами, а не запятыми. Для CSV нужен явный `awk -F ","`.
> - **`uniq` без `sort`.** `uniq` сравнивает только *соседние* строки. Если повторы разбросаны по файлу, без предварительной сортировки они останутся: `cat names.txt | uniq` даст совсем не тот результат, что `sort names.txt | uniq`.
> - **`grep -E` против `grep`.** Базовый `grep` понимает только ограниченный набор регулярных выражений: `+`, `?`, `|` и группы из них требуют `grep -E` (либо экранирования). Если шаблон «не ищет» — первая мысль: «а это точно базовый regex или расширенный?».
> - **Перенаправление в тот же файл.** `sed 's/.../.../' file.txt > file.txt` обнулит `file.txt`: оболочка открывает файл на запись *до* запуска `sed`, и тот читает уже пустой файл. Для замены «на месте» есть `sed -i`.

## Разобранный пример: топ-5 IP в логе nginx

**Задача.** В access-логе nginx (`/var/log/nginx/access.log`) первой колонкой идёт IP-адрес клиента. Нужно вывести пять самых активных IP с количеством запросов.

**Решение.**

1. Извлечь первый столбец из лога. Поля разделены пробелами, поэтому `awk` сработает без `-F`:
   ```shell
   $ awk '{ print $1 }' /var/log/nginx/access.log
   ```
2. Сгруппировать одинаковые адреса. `uniq -c` считает повторы, но требует, чтобы повторы шли подряд, — значит, сначала `sort`:
   ```shell
   $ awk '{ print $1 }' /var/log/nginx/access.log | sort | uniq -c
   ```
3. Отсортировать по убыванию числа запросов. Флаг `-n` сравнивает как числа, `-r` переворачивает порядок:
   ```shell
   $ awk '{ print $1 }' /var/log/nginx/access.log | sort | uniq -c | sort -nr
   ```
4. Оставить только первые пять строк:
   ```shell
   $ awk '{ print $1 }' /var/log/nginx/access.log | sort | uniq -c | sort -nr | head -n 5
   ```

**Проверка.** Команда печатает пять строк, в каждой — число запросов и IP-адрес, отсортированные по убыванию. Сравните сумму чисел с `wc -l /var/log/nginx/access.log`: пятёрка лидеров должна давать заметную долю общего числа запросов.

## `sed`

Если `grep` отбирает строки, а `awk` работает с колонками, то `sed` (stream editor) — это инструмент для преобразования текста на лету: он пропускает поток через себя и по дороге что-то меняет. Чаще всего его используют для замены.

Заменить первое совпадение в каждой строке:

```shell
$ sed 's/error/ERROR/' app.log
```

Команда `s/что/на что/` — это «найти и заменить». Без дополнительных указаний `sed` заменяет только первое совпадение в строке; чтобы заменить все, добавляют флаг `g` (global):

```shell
$ sed 's/error/ERROR/g' app.log
```

`sed` умеет не только заменять, но и удалять строки. Например, выражение `/^$/d` означает «удалить пустые строки» (`^$` — строка, в которой между началом и концом ничего нет):

```shell
$ sed '/^$/d' notes.txt
```

## `xargs`

Иногда вывод одной команды нужно отдать другой не как текст на вход, а как аргументы. Например, `find` выдаёт список файлов построчно, а `ls -l` ждёт имена файлов как аргументы — напрямую через `|` это не соединить. Мостиком служит `xargs`: он берёт строки со входа и подставляет их аргументами в указанную команду.

```shell
$ cat files.txt | xargs ls -l
$ find . -name "*.log" | xargs wc -l
```

Есть один подвох: по умолчанию `xargs` разделяет аргументы по пробелам, поэтому имя файла с пробелом развалится на две части. Надёжное решение — попросить `find` разделять имена не переводом строки, а нулевым байтом (`-print0`), а `xargs` — ожидать именно такой разделитель (`-0`):

```shell
$ find . -name "*.log" -print0 | xargs -0 wc -l
```
