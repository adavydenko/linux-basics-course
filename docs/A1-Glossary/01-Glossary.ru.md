# Глоссарий

Краткие определения терминов, встречающихся в пособии. Если термин одинаково часто встречается на русском и английском, основная запись — по русскому варианту, в скобках указано исходное английское слово.

## А

**Алиас (alias)** — собственное короткое имя для команды или цепочки команд, заведённое через `alias`. Живёт только в текущей сессии оболочки.

**Аргумент команды** — значение, которое команда обрабатывает. Например, в `cp file.txt /tmp` аргументы — это `file.txt` и `/tmp`. Не путать с опцией.

**Аутентификация** — проверка того, кто именно пытается войти. В Linux чаще всего по паролю или по SSH-ключу.

## Б

**Bash (Bourne Again Shell)** — самая распространённая оболочка Linux; команда по умолчанию для интерактивной работы в большинстве дистрибутивов.

**BIOS / UEFI** — встроенная программа на материнской плате, которая запускается сразу после включения компьютера и передаёт управление загрузчику.

**Блочное устройство (block device)** — файл в `/dev`, представляющий накопитель или раздел. Чтение и запись идут блоками фиксированного размера. Отличается от обычных файлов буквой `b` в первой колонке `ls -l`.

## В

**Виртуальная машина (VM)** — программная эмуляция отдельного компьютера: VirtualBox, VMware, KVM, multipass. Внутри VM работает полноценная гостевая операционная система.

**Volume Group (VG)** — в терминологии LVM группа томов, общий «пул» дискового пространства, собранный из одного или нескольких physical volumes.

**Volume Group, Physical Volume, Logical Volume** — три уровня LVM. См. отдельные статьи.

## Г

**Gateway (шлюз)** — маршрутизатор, на который система отправляет пакеты для сетей, не описанных в её таблице маршрутизации.

**GPT (GUID Partition Table)** — современная схема разметки диска, поддерживающая больше четырёх разделов и тома в петабайты. Преемник MBR.

**GID (Group ID)** — числовой идентификатор группы. Виден в выводе `id`, в третьем поле `/etc/group`.

**GRUB** — загрузчик операционной системы. После BIOS/UEFI именно GRUB загружает выбранное ядро Linux.

**Группа (group)** — именованный набор пользователей. Используется для разграничения прав на файлы.

## Д

**Daemon (демон)** — фоновая служба, работающая без терминала. Имена демонов обычно оканчиваются на `d`: `sshd`, `crond`, `systemd`.

**DAS (Direct Attached Storage)** — хранилище, подключённое напрямую к одному хосту (как обычный диск), без сети.

**DHCP** — протокол, по которому машина при загрузке автоматически получает IP-адрес, маску, шлюз и DNS-сервер.

**DNS (Domain Name System)** — распределённая система, переводящая доменные имена (`google.com`) в IP-адреса (`172.217.0.132`).

**Docker** — самая распространённая система контейнеризации. Использует образы и контейнеры (см.).

**Дескриптор файла (file descriptor, fd)** — небольшое целое число, по которому процесс обращается к открытому файлу или потоку. У каждого процесса по умолчанию открыты: `0` — stdin, `1` — stdout, `2` — stderr.

**DPKG (Debian Package Manager)** — низкоуровневый пакетный менеджер семейства Debian; работает с файлами `.deb`. APT — удобная надстройка над ним.

**DROP / REJECT** — действия iptables: `DROP` молча отбрасывает пакет, `REJECT` отвечает отправителю сообщением «соединение отклонено».

## E

**ed25519** — современный алгоритм генерации SSH-ключей. Короче и быстрее RSA при более высокой криптостойкости.

**Extended партиция (расширенный раздел)** — особый тип раздела в схеме MBR, который сам данные не хранит, а служит контейнером для логических разделов. Способ обойти ограничение MBR в 4 «основных» раздела.

## Ж

**Журнал (journal)** — единый журнал событий, который собирает `systemd-journald` в системах с systemd. Содержит сообщения ядра, всех служб и других источников. Читается через `journalctl`.

## З

**Загрузчик** — программа, которая после BIOS/UEFI загружает ядро операционной системы. В Linux чаще всего это GRUB.

## I

**ICMP** — протокол, по которому работают `ping` и `traceroute`. Не имеет портов, используется для служебных сообщений сети.

**`ip` (iproute2)** — современный набор команд для работы с сетью в Linux: `ip addr`, `ip link`, `ip route`. Заменил устаревшие `ifconfig` и `route` из пакета `net-tools`.

**iptables** — классический инструмент фильтрации пакетов в Linux. На современных дистрибутивах постепенно вытесняется `nftables`, `ufw` и `firewalld`.

**Inode** — внутренняя запись файловой системы о файле: его размер, владелец, права, дата изменения и список блоков данных. Имени файла в inode нет — оно лежит в каталоге как ссылка на inode.

**IP-адрес** — числовой адрес узла в сети. IPv4 записывается четырьмя числами через точку (`192.168.1.10`), IPv6 — шестнадцатеричной формой с двоеточиями.

## К

**Кэш** — временное хранилище недавно полученных данных, чтобы не запрашивать их повторно. Например, кэш DNS у браузера или у системного DNS-резолвера.

**Командный режим (vi)** — основной режим редактора vi, в котором клавиши работают как команды («удалить строку», «перейти в конец»), а не вводят текст.

**Контейнер (container)** — изолированный процесс, использующий ядро хоста, но с собственной файловой системой и сетью. Базовая идея Docker и Podman.

**Конвейер (pipe, `|`)** — соединение stdout одной команды со stdin следующей: `cat file | grep error | wc -l`.

**cgroups (control groups)** — механизм ядра, который ограничивает ресурсы (CPU, память, диск) для группы процессов. Используется контейнерами и systemd.

**chmod** — команда, меняющая права доступа к файлу (`u+x`, `750`).

**chown / chgrp** — команды смены владельца / группы файла.

**Cron** — служба, выполняющая команды по расписанию. Расписание задаётся в crontab.

**Crontab** — таблица заданий cron. Команда `crontab -e` открывает её для текущего пользователя.

## Л

**Лог** — текстовая запись о событии в системе. Логи живут либо в `/var/log/`, либо в systemd journal.

**Логический том (LV, Logical Volume)** — «виртуальный раздел», нарезанный из Volume Group. Именно на LV создают файловую систему и монтируют его.

**Loopback (lo)** — внутренний сетевой интерфейс машины. Адрес `127.0.0.1` всегда указывает на саму машину.

**LVM (Logical Volume Manager)** — менеджер логических томов. Прослойка между физическими дисками и файловыми системами, позволяющая собирать тома из нескольких дисков и менять их размер на ходу.

## М

**MAC-адрес** — аппаратный адрес сетевого интерфейса, шесть пар шестнадцатеричных цифр: `0e:d5:8a:af:00:de`.

**Major / minor number** — пара чисел у файлов блочных и символьных устройств. Major указывает на драйвер устройства, minor — на конкретный экземпляр.

**Маска подсети (subnet mask)** — определяет, какие IP-адреса считаются «соседями» в той же сети. Записывается либо через точки (`255.255.255.0`), либо коротко в CIDR (`/24`).

**Маршрут (route)** — запись в таблице маршрутизации: какие пакеты на какой шлюз отправлять. Маршрут «по умолчанию» (`default`) применяется ко всему, для чего нет более точного правила.

**Маршрутизатор (router)** — устройство, соединяющее разные сети между собой и пересылающее пакеты по маршрутам.

**Межсетевой экран (firewall)** — система фильтрации сетевого трафика. В Linux реализуется через iptables, nftables, ufw, firewalld.

**MBR (Master Boot Record)** — историческая схема разметки диска. Хранит до четырёх «основных» разделов и не поддерживает диски больше 2 ТБ.

**`mount`** — команда подключения файловой системы в дерево каталогов в указанную точку монтирования. Без аргументов выводит список смонтированных систем.

**Multi-user.target** — цель загрузки systemd: система загружена, многопользовательский режим, сеть работает. Стандартный режим серверов.

## Н

**Namespace (пространство имён)** — механизм ядра, изолирующий ресурсы (процессы, сеть, точки монтирования) для группы процессов. Используется контейнерами.

**NAS (Network Attached Storage)** — сетевое хранилище, отдающее данные как файлы и каталоги по протоколам вроде NFS или SMB/CIFS.

**Netplan** — современная система описания сетевой конфигурации в Ubuntu (с 17.10). YAML-файлы в `/etc/netplan/`.

**NetworkManager** — служба управления сетевыми соединениями, типичная для Ubuntu Desktop, RHEL, Fedora.

**NFS (Network File System)** — протокол сетевого файлового доступа, изначально из мира Unix. Сервер описывает экспортируемые каталоги в `/etc/exports`.

**nftables** — современная подсистема фильтрации пакетов в Linux, преемник iptables.

**nice / renice** — изменение приоритета процесса в диапазоне `-20…19` (меньше — выше приоритет).

## О

**Образ (image)** — шаблон контейнера. Контейнер создаётся как «запущенный экземпляр» образа.

**Опция команды (флаг)** — модификатор поведения команды, обычно начинается с дефиса: `ls -l`, `grep -i`.

**Основной раздел (primary partition)** — раздел в схеме MBR, способный нести загружаемую операционную систему. В MBR их не больше четырёх.

**Оболочка (shell)** — программа-посредник между пользователем и операционной системой. Принимает команды и возвращает результат. Самая распространённая — Bash.

## П

**Пакет (package)** — архив с приложением и описанием его зависимостей. В Debian-системах — `.deb`, в RPM-системах — `.rpm`.

**Пакетный менеджер** — утилита установки/удаления/обновления пакетов: `apt`, `dpkg`, `yum`, `dnf`, `rpm`.

**Перенаправление (redirection)** — направление stdin/stdout/stderr процесса в файл или другую команду: `>`, `>>`, `<`, `2>`, `2>&1`, `|`.

**PID (Process ID)** — числовой идентификатор процесса. Виден в `ps` и нужен `kill`/`renice`.

**Process substitution** — конструкция `<(...)` или `>(...)`, дающая команде вместо файла «псевдофайл», подключённый к выводу/вводу другой команды.

**PPID (Parent PID)** — PID родительского процесса. В Linux у каждого процесса (кроме `systemd` с PID 1) есть родитель.

**Точка монтирования (mount point)** — каталог в дереве файловой системы, в который смонтирована другая файловая система. В Linux нет «букв дисков» — все файлы видны как единое дерево.

**Приоритет процесса** — число `nice`, влияющее на распределение процессорного времени. См. `nice`, `renice`.

**Process** — см. процесс.

**Процесс** — экземпляр запущенной программы. У каждого процесса есть PID, владелец и состояние (R, S, D, Z).

**Прерывание Ctrl+C** — посылает процессу сигнал `SIGINT`, по которому большинство программ корректно завершаются.

**PV (Physical Volume)** — в терминологии LVM физический том: реальный диск или раздел, отданный под управление LVM.

**`PATH`** — переменная окружения со списком каталогов, в которых оболочка ищет исполняемые файлы.

**`PS1`** — переменная окружения, задающая внешний вид приглашения Bash. Поддерживает специальные подстановки вроде `\u`, `\h`, `\w`.

## Р

**RAID** — массив дисков с дублированием или распределением данных. В пособии не разбирается подробно, но упоминается рядом с LVM.

**Регулярное выражение (regex)** — формальный язык для описания шаблонов в тексте. Применяется в `grep`, `sed`, `awk`. См. приложение «Регулярные выражения».

**Репозиторий** — сетевой источник пакетов. APT берёт пакеты из репозиториев, описанных в `/etc/apt/sources.list` и `sources.list.d/`.

**Root (рут)** — особый пользователь с UID 0 и неограниченными правами. Прямой вход под root в современных Ubuntu обычно заблокирован — права получают через `sudo`.

**Runlevel** — историческое понятие уровня загрузки в SysV-init. В systemd ту же роль играют targets.

**RPM** — пакетный формат и одноимённая низкоуровневая утилита семейства RHEL/Fedora.

**RHEL (Red Hat Enterprise Linux)** — коммерческий дистрибутив Linux. На его основе собраны CentOS Stream, Rocky Linux, AlmaLinux.

## С

**SAN (Storage Area Network)** — выделенная сеть хранения, отдающая клиентам данные на уровне блоков (как локальный диск).

**`scp`** — копирование файлов между машинами по защищённому каналу SSH.

**`sed`** — потоковый редактор для построчной обработки текста. Часто используется с регулярными выражениями.

**Сессия (session)** — связка пользователь + терминал на время одной работы в системе. Закрытие терминала или SSH-соединения завершает сессию.

**Сигнал (signal)** — короткое уведомление, которое ядро или другой процесс посылает целевому процессу. Распространённые: `SIGINT` (Ctrl+C), `SIGTERM` (вежливое завершение), `SIGKILL` (принудительное), `SIGHUP` (перечитать конфигурацию).

**Системный вызов (syscall)** — запрос пользовательского процесса к ядру (`open`, `read`, `write`, `fork`). Граница между пользовательским и ядерным пространством.

**Skript / скрипт** — текстовый файл с командами оболочки, исполняемый построчно. Первая строка — shebang (`#!/bin/bash`).

**`ss` (socket statistics)** — современная замена `netstat`. Показывает сокеты, открытые порты и активные соединения.

**SSH (Secure Shell)** — протокол и команда для защищённой работы на удалённой машине по сети.

**`sudo`** — выполнение одной команды от имени другого пользователя (по умолчанию root). Спрашивает ваш собственный пароль, а не пароль root.

**`systemctl`** — главная команда управления службами systemd.

**`systemd`** — современная система инициализации Linux и менеджер служб. Первый процесс системы (PID 1).

**Stack trace** — список вложенных вызовов функций в момент ошибки. Читается с конца — там лежит само исключение и место сбоя.

**stdin / stdout / stderr** — стандартные потоки процесса: ввод, вывод и ошибки. Дескрипторы `0`, `1`, `2`.

**Switch (коммутатор)** — устройство, связывающее устройства внутри одной сети.

## Т

**Target (цель загрузки systemd)** — состояние, в которое systemd «выводит» систему. Примеры: `multi-user.target` (сервер), `graphical.target` (рабочий стол).

**Таблица маршрутизации (routing table)** — список маршрутов в ядре. Показывается через `ip route`.

**Таймер (systemd timer)** — unit-файл, запускающий другой unit по расписанию. Современная альтернатива cron.

**`tmux` / `screen`** — менеджеры терминальных сессий: запустил задачу — отключился — задача продолжает работать.

**Точка монтирования** — см. mount point.

**Traceroute** — диагностическая команда, выводящая каждый промежуточный маршрутизатор на пути до цели.

**TTL (Time To Live)** — поле IP-пакета, ограничивающее число хопов; используется и в DNS — там TTL это срок жизни ответа в кэше.

## У

**UID (User ID)** — числовой идентификатор пользователя. UID `0` — root.

**Юнит (systemd unit)** — единица управления systemd: служба (`.service`), точка монтирования (`.mount`), таймер (`.timer`), цель (`.target`).

**`ufw` (Uncomplicated Firewall)** — упрощённая надстройка над nftables/iptables, типичная для Ubuntu.

**Umask** — маска прав, которая обрезает права у вновь создаваемых файлов. По умолчанию `022`: новые файлы — `644`, новые каталоги — `755`.

**Учётная запись** — запись о пользователе в системе. Хранится в `/etc/passwd` и (для пароля) `/etc/shadow`.

**UUID (Universally Unique IDentifier)** — уникальный идентификатор объекта (файловой системы, диска, раздела). Используется в `/etc/fstab` вместо изменчивых имён вроде `/dev/sdb1`.

## Ф

**Файловая система (filesystem)** — структура, по которой данные на разделе организованы в файлы и каталоги. Семейство в Linux: ext2/3/4, XFS, Btrfs, F2FS.

**Fibre Channel** — оптическая технология построения SAN. В пособии не рассматривается подробно.

**Flag (флаг)** — синоним опции команды.

**Foreground / background** — выполнение процесса «на переднем плане» (с занятым терминалом) или «в фоне» (`&`). Управляется через `fg`, `bg`, `jobs`.

**fork** — системный вызов, порождающий дочерний процесс.

**FQDN (Fully Qualified Domain Name)** — полное доменное имя с указанием всех зон до корня: `host.example.com.`.

## Х

**Хост** — узел в сети: компьютер, сервер, виртуальная машина.

**Хэш пароля** — необратимое преобразование пароля. В Linux хранится в `/etc/shadow`.

## Ц

**Цели загрузки** — см. target.

## Ч

**Чтение / запись / выполнение (r/w/x)** — три права доступа Unix. Для файла: `r` — читать содержимое, `w` — изменять, `x` — запускать. Для каталога: `r` — видеть список, `w` — создавать/удалять файлы, `x` — входить в каталог.

## Ш

**Шлюз** — см. gateway.

**Шаблон оболочки (glob)** — упрощённый шаблон имени: `*` (любая последовательность), `?` (любой один символ), `[abc]` (один из перечисленных). Не путать с регулярным выражением.

## Я

**Ядро (kernel)** — центральная часть операционной системы. Управляет памятью, процессами, оборудованием, файловыми системами.

**Ядерное пространство / пользовательское пространство (kernel space / user space)** — две зоны выполнения. В ядерное переходят только через системные вызовы; пользовательские программы напрямую туда не лезут.

## A–Z (английские термины, не нашедшие места выше)

**API (Application Programming Interface)** — интерфейс прикладного программирования: набор функций, через которые программы общаются между собой.

**ASCII** — таблица соответствия символов и чисел от 0 до 127.

**`awk`** — текстовый процессор для построчной обработки с поддержкой полей и регулярных выражений.

**CIDR (Classless Inter-Domain Routing)** — запись подсети через косую черту: `192.168.1.0/24`.

**`curl`** — утилита для HTTP/HTTPS-запросов из командной строки.

**FHS (Filesystem Hierarchy Standard)** — стандарт стандартного назначения каталогов корня в Linux (`/etc`, `/usr`, `/var`, …).

**GECOS** — пятое поле в `/etc/passwd`, традиционно для имени пользователя и контактной информации.

**HTTP / HTTPS** — протоколы веб-обмена. HTTPS — HTTP поверх шифрованного TLS-канала.

**`journalctl`** — команда для чтения systemd-журнала.

**`kill`** — команда отправки сигнала процессу.

**`nohup`** — запуск процесса без привязки к терминалу (no hangup).

**`pgrep` / `pkill`** — поиск/убивание процессов по имени.

**Polkit / PAM** — системы аутентификации и проверки прав. В курсе упоминаются вскользь.

**`tar`** — утилита создания и распаковки архивов. Часто в связке с `gzip` (`tar.gz`).

**TCP / UDP** — транспортные протоколы. TCP с установкой соединения и подтверждением; UDP — без, быстрее.

**`venv`** — модуль Python для создания изолированных окружений.

**Wildcard** — обиходное название glob-шаблона.

**YUM / DNF** — пакетные менеджеры RPM-семейства. `dnf` — преемник `yum`, синтаксис в основном совместим.

**`zsh`** — альтернативная оболочка, популярная среди опытных пользователей (по умолчанию в macOS).
