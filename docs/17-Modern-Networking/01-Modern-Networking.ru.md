# Современная сеть в Linux

В старых материалах часто встречаются `ifconfig`, `route`, `netstat`. В современных Linux-системах основным набором инструментов является `iproute2`: команды `ip` и `ss`.

## Цели раздела

После раздела студент должен уметь:

- смотреть IP-адреса интерфейсов;
- смотреть маршруты;
- проверять открытые порты и соединения;
- тестировать HTTP-сервис через `curl`;
- понимать место `iptables`, `nftables`, `ufw` и `firewalld`.

## IP-адреса

```shell
$ ip addr
$ ip a
```

Посмотреть конкретный интерфейс:

```shell
$ ip addr show eth0
```

## Маршруты

```shell
$ ip route
```

Маршрут по умолчанию обычно показывает, через какой gateway система выходит в другие сети.

## Соединения и порты

`ss` заменяет многие сценарии `netstat`.

```shell
$ ss -tuln
$ ss -tunap
```

Опции:

- `-t` — TCP;
- `-u` — UDP;
- `-l` — listening sockets;
- `-n` — не преобразовывать адреса и порты в имена;
- `-p` — показать процесс.

## Проверка сервиса

```shell
$ curl http://localhost:8080
$ curl -I https://example.com
$ curl -v http://localhost:8080
```

`curl -I` показывает только заголовки, а `curl -v` помогает увидеть детали соединения.

## Firewall: краткий контекст

`iptables` важен исторически и всё ещё встречается. В современных системах часто используются:

- `nftables` — современная подсистема фильтрации пакетов;
- `ufw` — упрощённый firewall-интерфейс в Ubuntu;
- `firewalld` — распространён в RHEL/Fedora/CentOS.

Для базового курса важно понимать идею: firewall управляет тем, какие сетевые соединения разрешены или запрещены.

## Практическое задание

1. Выполните `ip addr` и найдите активный интерфейс.
2. Выполните `ip route` и найдите default route.
3. Запустите простой HTTP-сервер: `python3 -m http.server 8000`.
4. В другом терминале проверьте его через `curl`.
5. Найдите слушающий порт через `ss -tuln`.
