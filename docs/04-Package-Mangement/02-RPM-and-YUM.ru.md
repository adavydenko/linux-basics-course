# Пакетные менеджеры RPM и YUM

- [Доп. видео](https://kodekloud.com/topic/rpm-and-yum/)

В прошлом разделе мы выяснили, что в мире RHEL/CentOS/Fedora работает пара инструментов: низкоуровневый `rpm` и удобная надстройка над ним. Теперь посмотрим на эту пару вблизи — что именно делает каждый инструмент, где проходит граница между ними и почему на практике вы почти всегда будете обращаться к верхнему.

## Цели раздела

После раздела студент должен уметь:

- объяснить, что делает `rpm`;
- понимать ограничение `rpm`: он не решает зависимости автоматически;
- объяснить, зачем нужен `yum`;
- устанавливать, удалять, обновлять и искать пакеты через `yum`;
- смотреть настроенные репозитории.

## RPM

Начнём с фундамента. `rpm` (RPM Package Manager — исторически Red Hat Package Manager) — это базовый инструмент управления пакетами в RHEL и других RPM-based дистрибутивах. Файлы таких пакетов имеют расширение `.rpm`, и `rpm` умеет работать с ними напрямую.

### Работа с RPM

У `rpm` несколько основных режимов работы, и вместе они покрывают весь жизненный цикл пакета:

1. Установка.
2. Удаление.
3. Обновление.
4. Запрос информации.
5. Проверка.

| Режим | Что делает |
|---|---|
| Установка | поставить пакет |
| Удаление | удалить пакет |
| Обновление | обновить версию |
| Запрос (query) | посмотреть информацию о пакете |
| Проверка | сверить файлы пакета с системой |

Казалось бы, этого достаточно. Но у `rpm` есть существенное ограничение: он не разрешает зависимости. Если устанавливаемому пакету нужны другие пакеты, `rpm` не пойдёт их искать и скачивать — он просто сообщит об ошибке и остановится. Разбираться, что и откуда доставить, придётся вам вручную. Именно поэтому в реальной работе поверх `rpm` используют менеджер уровнем выше — `yum`, а в новых системах `dnf`.

## YUM

`yum` (Yellowdog Updater Modified) — свободный пакетный менеджер для RPM-based систем, и он закрывает как раз то, чего не хватает `rpm`.

Что делает `yum`:

- работает с RPM-пакетами;
- использует программные репозитории — каталоги пакетов, доступные по сети;
- хранит информацию о репозиториях в каталоге `/etc/yum.repos.d/`;
- описывает каждый репозиторий отдельным файлом с расширением `.repo`;
- под капотом всё равно опирается на `rpm`;
- и главное — автоматически разрешает зависимости.

То есть `yum` не заменяет `rpm`, а управляет им: сам находит нужные пакеты в репозиториях, выстраивает цепочку зависимостей и передаёт `rpm` уже готовый список на установку.

## Как YUM устанавливает пакет

Чтобы команды дальше не казались магией, разберём, что происходит за время одного `yum install`.

Сначала `yum` проверяет транзакцию: смотрит, не установлен ли пакет уже. Если нет — ищет его в репозиториях, перечисленных в `/etc/yum.repos.d/`.

Найдя пакет, `yum` разбирается с зависимостями: какие из нужных пакетов уже стоят в системе, какие надо доустановить, а какие — обновить.

![yum-repo](../images/ru_yum-repo.svg)

Результат этого анализа `yum` показывает в виде сводки транзакции — списка того, что будет установлено и обновлено. Здесь он останавливается и ждёт вашего подтверждения: нажмите `y`, чтобы продолжить. Если вы заранее уверены в результате (например, в скрипте), этот вопрос можно пропустить флагом `-y`. После подтверждения `yum` скачивает все нужные RPM-пакеты и устанавливает их.

Обновление устроено похоже. Команда `yum update <package>` обновляет один пакет; если он уже последней версии, `yum` честно сообщит, что менять нечего, и ничего не сделает.

## Частые команды

Теперь, понимая логику `yum`, пройдёмся по командам, которые вы будете набирать чаще всего.

Прежде чем что-то ставить, полезно знать, откуда `yum` берёт пакеты. Список всех настроенных репозиториев покажет:

```shell
$ yum repolist
```

Иногда вы знаете нужную команду, но не знаете, в каком пакете она лежит. `yum provides` отвечает именно на этот вопрос — какой пакет предоставляет, например, `scp`:

```shell
$ yum provides scp
```

Установка пакета — основная команда модуля:

```shell
$ yum install httpd
```

Если хотите, чтобы `yum` не задавал вопрос о подтверждении и сразу отвечал «да», добавьте `-y`:

```shell
$ yum install httpd -y
```

Удаление пакета — зеркальная операция:

```shell
$ yum remove httpd
```

Обновить один конкретный пакет:

```shell
$ yum update telnet
```

А без имени пакета `yum update` обновит сразу все пакеты в системе:

```shell
$ yum update
```

## Практическое задание

1. Выполните `yum repolist` и посчитайте настроенные репозитории.
2. Найдите, какой пакет предоставляет команду `scp`.
3. Сравните смысл команд `rpm -qa` и `yum repolist`.
4. Объясните, почему `yum install package -y` удобен в скриптах, но может быть опасен при ручной работе.
