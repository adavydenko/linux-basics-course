# Пакетные менеджеры RPM и YUM

- Перейти к [видеоуроку](https://kodekloud.com/topic/rpm-and-yum/)

В этом разделе мы подробнее рассмотрим `RPM` и `YUM`.

## Цели раздела

После раздела студент должен уметь:

- объяснить, что делает `rpm`;
- понимать ограничение `rpm`: он не решает зависимости автоматически;
- объяснить, зачем нужен `yum`;
- устанавливать, удалять, обновлять и искать пакеты через `yum`;
- смотреть настроенные репозитории.

## RPM

`RPM` расшифровывается как Red Hat Package Manager. Он используется в RHEL и других RPM-based дистрибутивах. Пакеты имеют расширение `.rpm`.

![rpm](../../images/rpm.PNG)

### Работа с RPM

У `rpm` есть несколько основных режимов работы:

1. Установка.
2. Удаление.
3. Обновление.
4. Запрос информации.
5. Проверка.

![rpm-modes](../../images/rpm-modes.PNG)

Главное ограничение `rpm`: он сам не разрешает зависимости. Поэтому на практике часто используют более высокий уровень управления пакетами — `yum` или в новых системах `dnf`.

## YUM

`YUM` расшифровывается как Yellowdog Updater Modifier. Это свободный пакетный менеджер для RPM-based систем.

`YUM`:

- работает с RPM-пакетами;
- использует программные репозитории;
- хранит информацию о репозиториях в `/etc/yum.repos.d/`;
- использует файлы репозиториев с расширением `.repo`;
- под капотом опирается на `rpm`;
- автоматически разрешает зависимости.

![yum](../../images/yum.PNG)

## Как YUM устанавливает пакет

Когда выполняется `yum install`, YUM сначала проверяет транзакцию. Если пакет не установлен, YUM ищет его в настроенных репозиториях из `/etc/yum.repos.d/`.

Затем он проверяет зависимости: какие пакеты уже установлены, какие нужно установить или обновить.

![yum-repo](../../images/yum-repo.PNG)

После проверки YUM показывает сводку транзакции. Пользователь может подтвердить установку клавишей `y`. Этот шаг можно пропустить флагом `-y`.

![yum-cmd](../../images/yum-cmd.PNG)

После подтверждения YUM скачивает и устанавливает нужные RPM-пакеты.

![yum-cmd1](../../images/yum-cmd1.PNG)

Если нужно обновить один пакет, используется `yum update <package>`. Если пакет уже последней версии, изменений не будет.

![yum-update](../../images/yum-update.PNG)

## Частые команды

Посмотреть все репозитории, добавленные в систему:

```shell
$ yum repolist
```

Проверить, какой пакет предоставляет нужную команду:

```shell
$ yum provides scp
```

Установить пакет:

```shell
$ yum install httpd
```

Установить пакет и автоматически отвечать `yes` на вопросы:

```shell
$ yum install httpd -y
```

Удалить пакет:

```shell
$ yum remove httpd
```

Обновить один пакет:

```shell
$ yum update telnet
```

Обновить все пакеты в системе:

```shell
$ yum update
```

## Практическое задание

1. Выполните `yum repolist` и посчитайте настроенные репозитории.
2. Найдите, какой пакет предоставляет команду `scp`.
3. Сравните смысл команд `rpm -qa` и `yum repolist`.
4. Объясните, почему `yum install package -y` удобен в скриптах, но может быть опасен при ручной работе.
