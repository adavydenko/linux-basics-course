# Пакетные менеджеры DPKG и APT

- Перейти к [видеоуроку](https://kodekloud.com/topic/dpkg-and-apt/)

В этом разделе мы рассмотрим пакетные менеджеры Debian-based дистрибутивов: Ubuntu, Debian, PureOS и похожих систем.

## Цели раздела

После раздела студент должен уметь:

- объяснить роль `dpkg`;
- понимать ограничение `dpkg`: он не решает зависимости автоматически;
- использовать `apt` для установки, обновления, удаления и поиска пакетов;
- понимать назначение `/etc/apt/sources.list`.

## DPKG

`DPKG` означает Debian Package Manager. Это низкоуровневый пакетный менеджер для пакетов `.deb`.

![dpkg](../../images/dpkg.PNG)

### Работа с DPKG

`dpkg` можно использовать для:

1. Установки.
2. Удаления.
3. Обновления.
4. Просмотра списка пакетов.
5. Проверки статуса.
6. Проверки пакета.

![dpkg-modes](../../images/dpkg-modes.PNG)

## APT и APT-GET

Как и `rpm`, `dpkg` сам не разрешает зависимости. Установка может завершиться ошибкой, если нужные пакеты не установлены.

![dpkg-fail](../../images/dpkg-fail.PNG)

Для удобной установки программ вместе с зависимостями используются более высокоуровневые инструменты: `apt` и `apt-get`.

```shell
$ sudo apt install gimp
$ sudo apt-get install gimp
```

`APT` выступает как frontend для управления пакетами и опирается на `dpkg`. По смыслу это похоже на связку `yum` и `rpm`: пользователь работает с более удобным инструментом, а низкоуровневую установку выполняет базовый менеджер.

Репозитории APT описываются в файле `/etc/apt/sources.list` и дополнительных файлах в `/etc/apt/sources.list.d/`.

![apt](../../images/apt.PNG)

## Частые команды

Обновить локальный список пакетов из репозиториев:

```shell
$ sudo apt update
```

Установить доступные обновления для установленных пакетов:

```shell
$ sudo apt upgrade
```

Открыть список источников пакетов в текстовом редакторе:

```shell
$ sudo apt edit-sources
```

Установить пакет:

```shell
$ sudo apt install telnet
```

Удалить пакет:

```shell
$ sudo apt remove telnet
```

Найти пакет в репозитории:

```shell
$ sudo apt search telnet
```

Вывести пакеты, связанные с `telnet`:

```shell
$ sudo apt list | grep telnet
```

## Практическое задание

1. Выполните `sudo apt update` и объясните, почему это не обновляет сами программы.
2. Найдите пакет `telnet` через `apt search`.
3. Посмотрите файл `/etc/apt/sources.list`.
4. Объясните разницу между `dpkg -i package.deb` и `apt install package`.
