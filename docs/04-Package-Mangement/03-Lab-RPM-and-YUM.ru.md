# Лабораторная работа: RPM и YUM

- Перейти к [лабораторной работе](https://kodekloud.com/topic/lab-yum-and-rpm/)

## Цели лабораторной работы

После выполнения работы студент должен уметь:

- выбрать подходящий пакетный менеджер для CentOS/RHEL;
- искать установленный пакет через `rpm`;
- понимать, почему установка `.rpm` напрямую может завершиться ошибкой зависимостей;
- устанавливать пакет через `yum`;
- смотреть репозитории и искать пакет, предоставляющий команду.

## Задания

1. Ответить, какие пакетные менеджеры используются на CentOS:

   ```text
   CentOS использует RPM-пакеты и YUM/DNF для управления зависимостями.
   ```

2. Найти точное имя установленного пакета `wget` через `rpm`:

   ```shell
   $ rpm -qa | grep wget
   ```

3. Установить пакет браузера `firefox`, скачанный в `/home/bob`. Возможна ошибка зависимостей:

   ```shell
   $ sudo rpm -ivh /home/bob/firefox-68.6.0-1.el7.centos.x86_64.rpm
   ```

4. Установить `firefox` вместе с зависимостями через `yum`:

   ```shell
   $ sudo yum install firefox -y
   ```

5. Проверить, сколько репозиториев YUM настроено в системе:

   ```shell
   $ sudo yum repolist
   ```

6. Проверить, какой пакет предоставляет команду `tcpdump`:

   ```shell
   $ sudo yum provides tcpdump
   ```

## Что нужно сдать

- Вывод `rpm -qa | grep wget`.
- Результат `sudo yum repolist`.
- Команду установки `firefox` через `yum`.
- Объяснение, почему установка через `rpm` может не сработать из-за зависимостей.

## Контрольные вопросы

1. Чем `rpm -ivh` отличается от `yum install`?
2. Почему `yum provides` полезен?
3. Что показывает `yum repolist`?
4. Когда можно использовать флаг `-y`?
