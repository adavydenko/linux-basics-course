# Лабораторная работа: RPM и YUM

- [Доп. лабораторная](https://kodekloud.com/topic/lab-yum-and-rpm/)

## Цели лабораторной работы

После выполнения работы студент должен уметь:

- выбрать подходящий пакетный менеджер для CentOS/RHEL;
- искать установленный пакет через `rpm`;
- понимать, почему установка `.rpm` напрямую может завершиться ошибкой зависимостей;
- устанавливать пакет через `yum`;
- смотреть репозитории и искать пакет, предоставляющий команду.

> **Окружение.** Лабораторную работу нужно выполнять на системе семейства RHEL (CentOS, RHEL, Rocky Linux, Fedora). Учебный Docker-контейнер из раздела «Настройка окружения» построен на Ubuntu — для этой лабораторной поднимите отдельный контейнер с CentOS 7/8 или RHEL/Rocky Linux. Имя пакета `firefox-68.6.0-1.el7.centos.x86_64.rpm` ниже — учебный пример из репозитория CentOS 7; в свежих сборках Firefox версия будет другой.

## Задания

В этой работе вы пройдёте типичный путь администратора RPM-системы: сначала определитесь с инструментами, затем заглянете в уже установленные пакеты, столкнётесь с ограничением `rpm` на живом примере и решите проблему через `yum`.

1. Определите, какие пакетные менеджеры используются на CentOS:

   ```text
   CentOS использует RPM-пакеты и YUM/DNF для управления зависимостями.
   ```

2. Узнайте точное имя установленного пакета `wget`. Пакет уже в системе, поэтому достаточно отфильтровать общий список через `grep`:

   ```shell
   $ rpm -qa | grep wget
   ```

3. Установите браузер `firefox` из `.rpm`-файла, скачанного в `/home/bob`. Здесь вы наглядно увидите ограничение `rpm`: команда может завершиться ошибкой зависимостей, потому что разрешать их `rpm` не умеет:

   ```shell
   $ sudo rpm -ivh /home/bob/firefox-68.6.0-1.el7.centos.x86_64.rpm
   ```

4. Установите тот же `firefox` через `yum` — он сам найдёт и доставит недостающие зависимости. Флаг `-y` сразу подтверждает транзакцию:

   ```shell
   $ sudo yum install firefox -y
   ```

5. Посмотрите, сколько репозиториев настроено в системе — именно из них `yum` брал пакеты на предыдущем шаге:

   ```shell
   $ sudo yum repolist
   ```

6. Выясните, какой пакет предоставляет команду `tcpdump` — приём, который пригодится, когда нужна команда, а имя её пакета неизвестно:

   ```shell
   $ sudo yum provides tcpdump
   ```
