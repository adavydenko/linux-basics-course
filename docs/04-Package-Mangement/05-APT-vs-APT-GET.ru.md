# APT и APT-GET

- Перейти к [видеоуроку](https://kodekloud.com/topic/apt-vs-apt-get/)

## Цели раздела

После раздела студент должен уметь:

- объяснить разницу между `apt` и `apt-get`;
- понимать, почему `apt` удобнее для интерактивной работы;
- знать, когда в инструкциях может встречаться `apt-get`.

## Основная разница

`apt` — более удобный для пользователя инструмент по сравнению с `apt-get`. В современных Debian-based дистрибутивах `apt` обычно установлен по умолчанию.

`apt-get` остаётся полезным и часто встречается в старых инструкциях, скриптах и Dockerfile. Но для ручной работы в терминале студентам обычно проще использовать `apt`.

## Пример установки

Попробуем установить пакет `firefox` через `apt` и `apt-get`.

При использовании `apt` вывод обычно компактнее, понятнее и содержит удобный индикатор прогресса. `apt-get` тоже эффективен, но его вывод менее дружелюбен для интерактивного пользователя.

## Пример поиска

Для поиска пакета `telnet` через `apt` используется:

```shell
$ apt search telnet
```

У `apt-get` нет команды `search`; для поиска нужно использовать отдельный инструмент:

```shell
$ apt-cache search telnet
```

Вывод `apt-cache` может содержать больше технических подробностей, которые не всегда нужны начинающему пользователю.

![apt-vs-apt-get1](../../images/apt-vs-apt-get1.PNG)

## Практическое правило

- Для ручной работы в терминале используйте `apt`.
- Если читаете старую инструкцию или скрипт, вы можете встретить `apt-get`.
- В автоматизации важно следовать стилю проекта и проверять, какой инструмент ожидается в окружении.
