# Основы сети: коммутация и маршрутизация

- [Доп. видео](https://kodekloud.com/topic/networking-basics/)

Чтобы пакет дошёл от одной машины до другой, его кто-то должен направить. Внутри одной сети это делает коммутатор (switch), а между разными сетями — маршрутизатор (router). Эти два устройства — основа всей сетевой связности, и в Linux с обоими можно работать из командной строки. Разберём, как посмотреть сетевые интерфейсы, назначить им адреса и настроить маршруты.

## Цели раздела

После раздела студент должен уметь:

- понимать разницу между switching и routing;
- смотреть сетевые интерфейсы;
- назначать IP-адрес интерфейсу;
- смотреть таблицу маршрутизации;
- добавлять маршрут через gateway.

## Switching

Коммутация (switching) работает в пределах одной сети: коммутатор связывает между собой устройства, у которых адреса из общего диапазона. Друг друга они «видят» напрямую, без посредников.

![switch](../images/ru_switch.svg)

Чтобы машина могла участвовать в сети, у неё должен быть сетевой интерфейс. Посмотреть список интерфейсов хоста поможет `ip link`:

```shell
$ ip link
eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP mode DEFAULT group default qlen 1000
```

В этой строке важны два момента. `eth0` — имя интерфейса, а `state UP` означает, что он включён и готов передавать данные (был бы `DOWN` — связи бы не было). Сам по себе интерфейс адреса не имеет — его нужно назначить (для правки сетевой конфигурации нужны права администратора):

```shell
$ sudo ip addr add 192.168.1.10/24 dev eth0
```

Запись `/24` после адреса задаёт маску подсети — она определяет, какие адреса считаются «соседями в той же сети», а какие — чужими.

## Routing

Внутри одной сети коммутатора достаточно. Но как только пакету нужно попасть в другую сеть, нужен маршрутизатор (router) — устройство, которое соединяет разные сети между собой.

![route](../images/ru_routing.svg)

Какие сети куда вести — система хранит в таблице маршрутизации. Исторически её смотрели командой `route` из пакета `net-tools`:

```shell
$ route
Kernel IP routing table
Destination Gateway Genmask Flags Metric Ref Use Iface
```

В современных дистрибутивах (Ubuntu 22.04, Debian 11+, RHEL 9+) пакет `net-tools` не установлен по умолчанию — команда просто скажет «command not found». Её современная замена — `ip route` из пакета `iproute2`, и пользоваться лучше именно ей:

```shell
$ ip route
```

Добавим в таблицу маршрут: чтобы добраться до сети `192.168.2.0/24`, пакеты нужно отправлять через шлюз (gateway) `192.168.1.1`. Изменение таблицы маршрутизации требует прав администратора:

```shell
$ sudo ip route add 192.168.2.0/24 via 192.168.1.1
```

Проверим, что маршрут появился:

```shell
$ route
Kernel IP routing table
Destination Gateway Genmask Flags Metric Ref Use Iface
192.168.2.0 192.168.1.1 255.255.255.0 UG 0 0 0 eth0
```

Читается строка так: трафик для сети `192.168.2.0` (`Destination`) отправляется через шлюз `192.168.1.1` (`Gateway`) и уходит с интерфейса `eth0` (`Iface`). Флаги `UG` означают, что маршрут активен (`U`, up) и ведёт через шлюз (`G`, gateway).

А посмотреть, какие адреса уже назначены интерфейсам, можно командой `ip addr`:

```shell
$ ip addr
```

Важный нюанс: всё, что мы настроили командами `ip`, действует только до перезагрузки. Чтобы изменения сохранялись, их записывают в конфигурацию сети. В старых Debian/Ubuntu это был файл `/etc/network/interfaces`; в современных системах эту роль взяли на себя NetworkManager, Netplan или systemd-networkd.

## Практическое задание

1. Выполните `ip link`.
2. Выполните `ip addr`.
3. Выполните `ip route`.
4. Найдите интерфейс с активным IP-адресом.
5. Объясните, какой маршрут используется по умолчанию.
