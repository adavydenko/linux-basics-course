# Процессы и ресурсы

Linux часто используется для долгих вычислений и серверных задач. Поэтому важно понимать, как запускать процессы, смотреть их состояние, завершать зависшие программы и оценивать нагрузку на систему.

## Цели раздела

После раздела студент должен уметь:

- просматривать процессы;
- запускать команды в фоне;
- возвращать фоновые задачи на передний план;
- завершать процессы;
- смотреть CPU, память и диск;
- запускать долгие задачи так, чтобы они не пропали после закрытия терминала.

## Процессы

Посмотреть процессы текущего пользователя:

```shell
$ ps
```

Посмотреть больше информации:

```shell
$ ps aux
$ ps aux | grep python
```

Интерактивный просмотр нагрузки:

```shell
$ top
```

Если установлен `htop`, он обычно удобнее:

```shell
$ htop
```

## Фоновые задачи

Запустить команду в фоне:

```shell
$ sleep 60 &
```

Посмотреть фоновые задачи текущего shell:

```shell
$ jobs
```

Вернуть задачу на передний план:

```shell
$ fg %1
```

Продолжить остановленную задачу в фоне:

```shell
$ bg %1
```

## Завершение процессов

Завершить процесс по PID:

```shell
$ kill 12345
```

Принудительное завершение:

```shell
$ kill -9 12345
```

`kill -9` стоит использовать только когда обычный `kill` не помогает: он не даёт программе корректно завершиться.

## Приоритеты

Запустить команду с меньшим приоритетом:

```shell
$ nice -n 10 long-task.sh
```

Изменить приоритет уже запущенного процесса:

```shell
$ renice 10 -p 12345
```

## Долгие задачи

`nohup` позволяет процессу продолжить работу после закрытия сессии:

```shell
$ nohup ./long-task.sh > out.log 2> err.log &
```

Для интерактивной долгой работы удобны `tmux` или `screen`:

```shell
$ tmux
$ ./long-task.sh
```

Отключиться от `tmux`: `Ctrl+b`, затем `d`.

Вернуться:

```shell
$ tmux attach
```

## Ресурсы системы

Память:

```shell
$ free -h
```

Диск:

```shell
$ df -h
$ du -sh *
```

Время работы и средняя нагрузка:

```shell
$ uptime
```

## Практическое задание

1. Запустите `sleep 300 &`.
2. Найдите процесс через `ps`.
3. Посмотрите задачу через `jobs`.
4. Верните её на передний план через `fg`.
5. Остановите её и продолжите в фоне.
6. Завершите процесс через `kill`.
7. Запустите тестовую долгую команду через `nohup` с отдельными файлами stdout и stderr.
