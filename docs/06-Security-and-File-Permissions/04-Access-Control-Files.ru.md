# Файлы управления доступом

- [Доп. видео](https://kodekloud.com/topic/access-control-files/)

Мы уже не раз заглядывали в `/etc/passwd` и `/etc/shadow`, чтобы что-то проверить. Пора рассмотреть эти файлы внимательно: что записано в каждом поле и почему именно они определяют, кто и как может войти в систему. Все эти файлы лежат в каталоге `/etc`. Большинство из них может читать широкий круг пользователей, а вот изменять — только `root`: ошибка в любом из них способна закрыть вход вообще всем.

## Цели раздела

После раздела студент должен уметь:

- читать структуру `/etc/passwd`;
- понимать назначение `/etc/shadow`;
- читать структуру `/etc/group`;
- объяснить, почему эти файлы критичны для безопасности.

## `/etc/passwd`

Файл `/etc/passwd` — это «телефонный справочник» системы: по строке на каждого пользователя, поля разделены двоеточиями. Вытащим строку про `bob` и поставим под ней названия полей:

```shell
$ grep -i ^bob /etc/passwd
bob:x:1002:1002::/home/bob:/bin/sh
USERNAME:PASSWORD:UID:GID:GECOS:HOMEDIR:SHELL
```

Читается строка так: имя пользователя `bob`, затем поле пароля, UID `1002`, GID `1002`, пустое поле GECOS (комментарий), домашний каталог `/home/bob` и оболочка `/bin/sh`. Самое любопытное здесь — буква `x` на месте пароля. Она не значит, что пароль равен «x». Это метка: «настоящий хэш пароля лежит не тут, а в `/etc/shadow`». Так сделано не случайно — об этом ниже.

## `/etc/shadow`

Когда-то хэши паролей хранились прямо в `/etc/passwd`. Но этот файл должны читать многие программы, а значит, и хэши были доступны всем — их можно было спокойно скопировать и подбирать пароль офлайн. Поэтому пароли вынесли в отдельный файл `/etc/shadow`, который читает только `root`:

```shell
$ sudo grep -i ^bob /etc/shadow
bob:$6$0h0utOtO$5JcuRxR7y72LLQk4Kdog7u09LsNFS0yZPkIC8pV9tgD0wXCHutYcWF/7.eJ3TfGfG0lj4JF63PyuPwKC18tJS.:18188:0:99999:7:::

USERNAME:PASSWORD:LASTCHANGE:MINAGE:MAXAGE:WARN:INACTIVE:EXPDATE
```

Здесь второе поле — это уже сам хэш пароля, а не метка. Остальные поля управляют сроком жизни пароля: когда он менялся в последний раз (`LASTCHANGE`), как часто его можно и нужно менять (`MINAGE`, `MAXAGE`), за сколько дней предупредить о смене (`WARN`) и когда учётная запись истекает.

## `/etc/group`

Третий файл — `/etc/group` — отвечает на вопрос «кто в каких группах состоит». Структура та же: поля через двоеточие:

```shell
$ grep -i ^bob /etc/group
NAME:PASSWORD:GID:MEMBERS
```

Поля по порядку: имя группы, поле пароля группы (используется редко), её GID и список участников. Именно через членство в группах проще всего давать нескольким пользователям общий доступ к файлам — это пригодится в следующем разделе про права.

## Практическое задание

1. Найдите запись своего пользователя в `/etc/passwd`.
2. Определите UID, GID, домашний каталог и оболочку.
3. Найдите группы своего пользователя через `id` и `/etc/group`.
4. Попробуйте прочитать `/etc/shadow` без `sudo` и объясните результат.
