# IPtables

- Перейти к [видеоуроку](https://kodekloud.com/topic/iptables-introduction/)

В этом разделе мы изучим базовые команды `iptables`.

> В современных дистрибутивах всё чаще используются `nftables`, `ufw` или `firewalld`, но `iptables` всё ещё часто встречается в учебных материалах, старых системах и документации.

## Цели раздела

После раздела студент должен уметь:

- понимать таблицы и цепочки `iptables`;
- смотреть текущие правила;
- добавлять разрешающие и запрещающие правила;
- отличать `DROP` от `REJECT`;
- удалять правило по номеру строки;
- понимать назначение `INPUT`, `OUTPUT`, `FORWARD`.

## Таблицы и цепочки

`iptables` использует таблицы. В таблицах есть цепочки, а цепочки содержат встроенные или пользовательские правила.

Основные таблицы:

1. `FILTER` — таблица по умолчанию. В ней есть цепочки:
   - `INPUT` — пакеты, предназначенные локальной системе;
   - `FORWARD` — пакеты, маршрутизируемые через систему;
   - `OUTPUT` — пакеты, созданные локально.
2. `NAT` — таблица для преобразования адресов при создании нового соединения. В ней есть цепочки:
   - `PREROUTING` — изменение пакета сразу после получения;
   - `OUTPUT` — изменение локально созданных пакетов;
   - `POSTROUTING` — изменение пакетов перед отправкой наружу.

Установить `iptables` в Ubuntu:

```shell
$ sudo apt install iptables
```

Посмотреть правила:

```shell
$ sudo iptables -L
```

Пример вывода:

```text
Chain INPUT (policy ACCEPT)
target     prot opt source               destination

Chain FORWARD (policy ACCEPT)
target     prot opt source               destination

Chain OUTPUT (policy ACCEPT)
target     prot opt source               destination
```

## Разрешающие правила

Разрешить входящие подключения от IP `172.16.238.187` к портам `22` и `80`:

```shell
$ sudo iptables -A INPUT -p tcp -s 172.16.238.187 --dport 22 -j ACCEPT
$ sudo iptables -A INPUT -p tcp -s 172.16.238.187 --dport 80 -j ACCEPT
```

Опции:

- `-A` или `--append` добавляет правило в конец цепочки;
- `-s` или `--source` задаёт источник;
- `-j` или `--jump` задаёт действие;
- `-p` или `--protocol` задаёт протокол;
- `--dport` задаёт порт назначения;
- `--sport` задаёт порт источника.

Посмотреть правила:

```shell
$ sudo iptables -L
```

## Запрещающие правила

Запретить входящие подключения из любого источника, на любой порт и по любому протоколу:

```shell
$ sudo iptables -A INPUT -j DROP
```

Такое правило обычно добавляют после разрешающих правил. Порядок важен: `iptables` проверяет правила сверху вниз.

## `DROP` и `REJECT`

`DROP` и `REJECT` запрещают прохождение пакета, но ведут себя по-разному.

- `DROP` молча отбрасывает пакет и не отправляет ответ.
- `REJECT` отклоняет пакет и отправляет источнику сообщение об ошибке.

## Исходящий трафик

Заблокировать исходящий трафик на порт `80`:

```shell
$ sudo iptables -A OUTPUT -p tcp --dport 80 -j DROP
```

Разрешить HTTPS-соединение с сервера к `google.com`:

```shell
$ sudo iptables -I OUTPUT -p tcp -d google.com --dport 443 -j ACCEPT
```

`-I` вставляет правило в начало цепочки, что важно, если ниже уже есть запрещающие правила.

## Удаление правила

Сначала нужно посмотреть номера строк:

```shell
$ sudo iptables -L --line-numbers
```

Удалить правило номер `3` из цепочки `INPUT`:

```shell
$ sudo iptables -D INPUT 3
```

## Несколько портов

Разрешить несколько портов через модуль `multiport`:

```shell
$ sudo iptables -A INPUT -p tcp -m multiport --dports 22,80,443 -j ACCEPT
$ sudo iptables -A OUTPUT -p tcp -m multiport --sports 22,80,443 -j ACCEPT
```

## ICMP и MAC

Заблокировать входящие ping-запросы на интерфейсе `eth0`:

```shell
$ sudo iptables -A INPUT -p icmp -i eth0 -j DROP
```

Заблокировать доступ с конкретного MAC-адреса:

```shell
$ sudo iptables -A INPUT -m mac --mac-source 0e:d5:8a:af:00:de -j DROP
```
