# SSH и SCP

- Перейти к [видеоуроку](https://kodekloud.com/topic/ssh-and-scp/)

В этом разделе мы рассмотрим команды `ssh` и `scp`.

`ssh` используется для входа на удалённый компьютер. `scp` используется для копирования файлов и каталогов, в том числе между локальной и удалённой машинами.

## Цели раздела

После раздела студент должен уметь:

- подключаться к удалённому серверу по SSH;
- указывать имя пользователя при подключении;
- понимать идею аутентификации по ключам;
- создавать пару ключей;
- копировать публичный ключ на сервер;
- копировать файлы и каталоги через `scp`.

## SSH

Подключиться к удалённому серверу по имени хоста или IP-адресу:

```shell
$ ssh <hostname OR IP Address>
```

Подключиться с конкретным именем пользователя:

```shell
$ ssh <user>@<hostname OR IP Address>
```

Также можно использовать опцию `-l`:

```shell
$ ssh -l <user> <hostname OR IP Address>
```

## Аутентификация без пароля

Вход без пароля настраивается через пару ключей: публичный и приватный ключ.

Публичный ключ:

```text
/home/bob/.ssh/id_rsa.pub
```

Приватный ключ:

```text
/home/bob/.ssh/id_rsa
```

Создать пару ключей на клиенте:

```shell
$ ssh-keygen -t rsa
```

![key](../../images//key.PNG)

Скопировать публичный ключ с клиента на удалённый сервер:

```shell
$ ssh-copy-id bob@devapp01
```

![copy](../../images//copy.PNG)

После этого Bob сможет входить на удалённый сервер без ввода пароля:

```shell
$ ssh devapp01
```

![pless](../../images//pless.PNG)

Публичный ключ сохраняется на удалённом сервере в `authorized_keys`:

```shell
$ cat /home/bob/.ssh/authorized_keys
```

![auth](../../images//auth.PNG)

Приватный ключ нельзя передавать другим людям или публиковать в репозитории.

## SCP

Скопировать сжатый файл на удалённый сервер:

```shell
$ scp /home/bob/caleston-code.tar.gz devapp01:/home/bob
```

Скопировать каталог на удалённый сервер:

```shell
$ scp -pr /home/bob/media/ devapp01:/home/bob
```
