# SSH и SCP

- [Доп. видео](https://kodekloud.com/topic/ssh-and-scp/)

Серверы, с которыми вы работаете, почти никогда не стоят рядом с вами — они в дата-центре или в облаке, за сотни километров. Значит, нужен способ безопасно работать с ними по сети: входить в систему и переносить файлы так, чтобы по пути никто не подсмотрел пароль или данные. Эту задачу решают две команды.

`ssh` (Secure Shell, «защищённая оболочка») открывает на удалённой машине терминал — будто вы сидите прямо за ней. `scp` (secure copy, «защищённое копирование») переносит файлы и каталоги между машинами. Обе шифруют трафик, поэтому им можно доверять даже в открытой сети.

## Цели раздела

После раздела студент должен уметь:

- подключаться к удалённому серверу по SSH;
- указывать имя пользователя при подключении;
- понимать идею аутентификации по ключам;
- создавать пару ключей;
- копировать публичный ключ на сервер;
- копировать файлы и каталоги через `scp`.

## SSH

Чтобы подключиться к серверу, `ssh` нужно знать только одно — куда идти. Адресом может быть имя хоста или IP-адрес:

```shell
$ ssh <hostname OR IP Address>
```

В таком виде `ssh` попробует войти под тем же именем, под которым вы работаете на своей машине. Но на сервере ваша учётная запись может называться иначе — тогда имя пользователя указывают явно, через символ `@`:

```shell
$ ssh <user>@<hostname OR IP Address>
```

То же самое можно записать и через опцию `-l` (login) — кому как удобнее:

```shell
$ ssh -l <user> <hostname OR IP Address>
```

## Аутентификация без пароля

Вводить пароль при каждом подключении неудобно, а хранить его в скриптах — небезопасно. Поэтому SSH предлагает другой способ входа — по паре ключей. Идея такая: у вас есть два связанных файла. **Приватный ключ** остаётся только на вашей машине и никогда её не покидает — это ваш секрет. **Публичный ключ** можно свободно раздавать; его кладут на серверы, куда вы хотите попадать. При подключении сервер проверяет: подходит ли ваш приватный ключ к публичному, который у него лежит. Подходит — вход разрешён, пароль не нужен.

По умолчанию ключи лежат в каталоге `.ssh` домашнего каталога. Публичный ключ:

```text
/home/bob/.ssh/id_ed25519.pub
```

Приватный ключ — заметьте, то же имя, но без расширения `.pub`:

```text
/home/bob/.ssh/id_ed25519
```

Создаётся пара ключей на вашей (клиентской) машине командой `ssh-keygen`. Опция `-t` задаёт алгоритм — современный выбор для новых ключей — `ed25519` (он короче и быстрее `rsa`, при этом криптостойкость выше):

```shell
$ ssh-keygen -t ed25519
```

Старые системы умеют только RSA (Rivest–Shamir–Adleman, исторически первый криптографический алгоритм с открытым ключом, названный по фамилиям авторов) — для них генерируют ключ командой `ssh-keygen -t rsa -b 4096`; в этом случае файлы будут называться `id_rsa` и `id_rsa.pub`.

Теперь публичный ключ нужно доставить на сервер. Можно сделать это вручную, но проще всего — командой `ssh-copy-id`: она сама подключится по паролю (в последний раз) и добавит ваш ключ куда надо:

```shell
$ ssh-copy-id bob@devapp01
```

После этого Боб входит на сервер без всякого пароля — SSH сам предъявляет приватный ключ:

```shell
$ ssh devapp01
```

На сервере публичный ключ оказывается в файле `authorized_keys` — это список всех ключей, которым разрешён вход:

```shell
$ cat /home/bob/.ssh/authorized_keys
```

И главное правило: приватный ключ никому не передают, не выкладывают в репозиторий и не копируют на чужие машины. Тот, у кого окажется ваш приватный ключ, сможет входить от вашего имени.

## SCP

Когда вход настроен, по тому же защищённому каналу удобно переносить файлы — этим занимается `scp`. Синтаксис похож на обычный `cp`, только удалённую сторону указывают как `сервер:путь`. Скопируем архив с локальной машины на сервер, в каталог `/home/bob`:

```shell
$ scp /home/bob/caleston-code.tar.gz devapp01:/home/bob
```

Каталог целиком одним файлом не скопируешь — нужна опция `-r` (recursive), которая идёт вглубь по вложенным каталогам. А `-p` (preserve) сохраняет у файлов время изменения и права доступа. Вместе получается `-pr`:

```shell
$ scp -pr /home/bob/media/ devapp01:/home/bob
```
