# Управление пользователями

- [Доп. видео](https://kodekloud.com/topic/user-management/)

В команду пришёл новый сотрудник — и ему нужна учётная запись. Кто-то ушёл — его доступ пора закрыть. Появился новый проект — для него удобно завести отдельную группу. Всё это рутинные задачи администратора, и в Linux они решаются небольшим набором команд. Разберём, как создавать и удалять пользователей и группы и как с самого начала задать пользователю нужные параметры.

## Цели раздела

После раздела студент должен уметь:

- создавать локального пользователя;
- проверять запись пользователя в `/etc/passwd`;
- менять пароль;
- задавать UID, GID, домашний каталог, оболочку и комментарий;
- удалять пользователя;
- создавать и удалять группы.

## Создание пользователя

Завести нового пользователя проще всего командой `useradd`. Она правит системные файлы `/etc/passwd` и `/etc/shadow`, поэтому требует прав администратора — отсюда `sudo`. В минимальном виде ей достаточно одного имени:

```shell
$ sudo useradd bob
```

Команда сработала молча — но что именно она сделала? Она добавила строку в `/etc/passwd`, назначив пользователю UID, GID, домашний каталог и оболочку по умолчанию. Убедимся в этом, вытащив нужную строку через `grep`:

```shell
$ grep -i bob /etc/passwd
bob:x:1002:1002::/home/bob:/bin/sh
```

Здесь видно: система выдала `bob` свободный UID `1002`, такой же GID, домашний каталог `/home/bob` и оболочку `/bin/sh`. А проверить, под кем вы работаете прямо сейчас, всегда можно командой `whoami` (who am I):

```shell
$ whoami
bob
```

Учётная запись создана, но войти под ней пока нельзя — у пользователя нет пароля. Сами пароли (точнее, их хэши) хранятся не в `/etc/passwd`, а в защищённом файле `/etc/shadow`, который обычному пользователю читать нельзя — поэтому и нужен `sudo`:

```shell
$ sudo grep -i bob /etc/shadow
bob:!:18341:0:99999:7:::
```

Восклицательный знак на месте пароля означает, что пароль ещё не задан и вход заблокирован. Зададим его командой `passwd`. Без аргумента она меняет пароль того, кто её запустил, и `sudo` тут не нужен. А вот чтобы задать или сбросить пароль другому пользователю, требуются права администратора:

```shell
$ sudo passwd bob
Changing password for user bob.
New UNIX password:
Retype new UNIX password:
passwd: all authentication tokens updated successfully.
```

Строка `all authentication tokens updated successfully` подтверждает, что пароль принят, — теперь под `bob` можно входить.

## Управление пользователями

Значения по умолчанию подходят не всегда. Часто хочется сразу задать пользователю конкретный UID, нужную группу, особый домашний каталог или удобную оболочку. Все эти параметры передаются `useradd` через опции в одной команде:

```shell
$ sudo useradd -u 1009 -g 1009 -d /home/robert -s /bin/bash -c "Mercury Project member" bob
```

Разберём опции:

- `-u` задаёт UID;
- `-g` задаёт основную группу;
- `-d` задаёт домашний каталог;
- `-s` задаёт оболочку;
- `-c` задаёт комментарий (его ещё называют полем GECOS — обычно сюда пишут имя сотрудника или назначение учётной записи).

Когда учётная запись больше не нужна, её удаляют командой `userdel`:

```shell
$ sudo userdel bob
```

Учтите: сама по себе эта команда убирает запись пользователя, но его домашний каталог остаётся на диске. Чтобы удалить и его, используют опцию `-r`.

Группы устроены проще. Создать новую группу с заданным GID:

```shell
$ sudo groupadd -g 1011 developer
```

И удалить ненужную:

```shell
$ sudo groupdel developer
```
