# История: инцидент безопасности

- Перейти к [истории](https://kodekloud.com/topic/security-incident/)

Однажды утром команда обнаруживает, что в системе что-то не так: кто-то получил доступ к файлам, к которым доступа быть не должно. Расследование начинается не с антивируса и не с хитрых утилит, а с простых вопросов — кто вошёл в систему, под какой учётной записью работал, какие файлы мог прочитать. Именно из этих вопросов и вырастает весь раздел.

Дальше вас ждут пользователи и группы, права доступа к файлам, удалённый вход по SSH и базовая сетевая фильтрация. Запомните главную мысль: в Linux безопасность держится не на одной большой защите, а на множестве мелких — кто ты, что тебе разрешено и до каких файлов ты можешь дотянуться.
