# Cron в Linux

- Перейти к [видеоуроку](https://kodekloud.com/courses/873064/lectures/24032686)

В этом разделе мы разберём `cron` — механизм запуска задач по расписанию.

## Цели раздела

После раздела студент должен уметь:

- объяснить, что такое `cron` и `crontab`;
- читать формат расписания crontab;
- редактировать список задач;
- понимать специальные строки вроде `@reboot`;
- составлять простые расписания.

`cron` используется для запуска команды в заданное время. `crontab` — это список команд, которые нужно выполнять по расписанию, а также имя команды для управления этим списком. `Crontab` означает `cron table`.

## Формат crontab

![format](../../images//cronformat.png)

Обычно строка crontab состоит из пяти полей времени и команды:

```text
minute hour day-of-month month day-of-week command
```

## Команды crontab

![command](../../images//croncmd.png)

Типовые команды:

```shell
$ crontab -l
$ crontab -e
$ crontab -r
```

## Примеры

```cron
*/30 * * * * command
```

Каждые 30 минут.

```cron
0 * * * * command
```

Каждый час.

```cron
0 0 * * 0 command
```

В полночь каждое воскресенье.

```cron
0 0 15 * * command
```

В полночь 15-го числа каждого месяца.

```cron
0 0 1 1 * command
```

В полночь 1 января.

```cron
@reboot command
```

При каждой загрузке системы.

## Практическое задание

1. Посмотрите текущий crontab через `crontab -l`.
2. Добавьте задачу, которая раз в минуту пишет дату в файл.
3. Проверьте результат через несколько минут.
4. Удалите тестовую задачу.
