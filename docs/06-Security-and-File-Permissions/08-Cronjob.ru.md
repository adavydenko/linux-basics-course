# Cron в Linux

- [Доп. видео](https://kodekloud.com/courses/873064/lectures/24032686)

Резервная копия каждую ночь, очистка временных файлов по выходным, отчёт первого числа месяца — есть задачи, которые нужно выполнять регулярно и в одно и то же время. Сидеть и ждать нужного момента, чтобы запустить команду руками, бессмысленно. В Linux за это отвечает `cron` — служба, которая по расписанию сама запускает заданные команды.

## Цели раздела

После раздела студент должен уметь:

- объяснить, что такое `cron` и `crontab`;
- читать формат расписания crontab;
- редактировать список задач;
- понимать специальные строки вроде `@reboot`;
- составлять простые расписания.

Сам `cron` — это фоновая служба, которая постоянно работает и раз в минуту сверяется со своим расписанием: не пора ли что-нибудь запустить. А расписание хранится в `crontab` (от `cron table`, «таблица cron»). Этим же словом называют и команду, которой это расписание правят, — так что `crontab` одновременно и список задач, и инструмент управления им.

## Формат crontab

![format](../images/ru_cronformat.svg)

Каждая задача в crontab — это одна строка. Сначала идут пять полей, которые отвечают на вопрос «когда», а затем — команда, которая отвечает на вопрос «что запустить»:

```text
minute hour day-of-month month day-of-week command
```

Пять полей по порядку: минута, час, день месяца, месяц и день недели. Звёздочка `*` в любом поле означает «любое значение» — то есть «всегда». Читать строку проще всего слева направо: например, `0 3 * * *` — это «в нулевую минуту третьего часа, в любой день, в любой месяц, в любой день недели», то есть каждую ночь в 3:00.

Допустимые значения и спецсимволы для каждого поля:

| Поле | Диапазон | Допустимые спецсимволы |
|---|---|---|
| Минута | 0–59 | `, - * /` |
| Час | 0–23 | `, - * /` |
| День месяца | 1–31 | `, - * ? / L W` |
| Месяц | 1–12 | `, - * /` |
| День недели | 0–6 | `, - * ? / L #` |

## Специальные выражения

Кроме обычных чисел в полях допускаются специальные символы — диапазоны, списки и шаги, — а часть распространённых расписаний можно записать готовыми ключевыми словами вроде `@daily`.

### Спецсимволы в полях

| Символ | Что значит |
|---|---|
| `*` (звёздочка) | любое значение в этом поле |
| `-` (дефис) | диапазон. Например, `1-5` в поле «День недели» — с понедельника по пятницу |
| `/` (слэш) | шаг. Например, `*/15` в поле «Минута» — каждые 15 минут |
| `,` (запятая) | перечисление. Например, `2,6,8` в поле «Час» — в 2:00, 6:00 и 8:00 |
| `L` | «последний». Допускается только в полях «День месяца» и «День недели»; `2L` в поле дня недели — «последний вторник месяца» |
| `#` | N-й день недели. Допускается только в поле «День недели»; `4#1` — «первый четверг месяца» |
| `?` | «любой», как `*`. Допускается только в полях «День месяца» и «День недели», причём ровно в одном из этих двух полей — одновременно использовать нельзя |

### Шаблоны-сокращения

Несколько распространённых расписаний можно записать одним ключевым словом вместо пяти полей:

| Выражение | Когда запускает | Эквивалент |
|---|---|---|
| `@reboot` | при каждой загрузке системы | — |
| `@yearly` (или `@annually`) | раз в год | `0 0 1 1 *` |
| `@monthly` | раз в месяц | `0 0 1 * *` |
| `@weekly` | раз в неделю | `0 0 * * 0` |
| `@daily` (или `@midnight`) | раз в день | `0 0 * * *` |
| `@hourly` | раз в час | `0 * * * *` |

Эти сокращения — расширение оригинального cron, поэтому они работают в большинстве современных дистрибутивов Linux, но формально не входят в стандарт POSIX.

## Команды crontab

Управлять своим расписанием вы будете несколькими командами:

| Команда | Что делает |
|---|---|
| `crontab -e` | редактировать crontab (создаст файл, если его ещё нет) |
| `crontab -l` | показать содержимое crontab |
| `crontab -r` | удалить crontab целиком |
| `crontab -v` | показать дату последней правки crontab — есть в Solaris/AIX, в Linux обычно **не реализована** |

С `crontab -r` будьте осторожны — она стирает расписание целиком и без подтверждения:

```shell
$ crontab -l
$ crontab -e
$ crontab -r
```

## Примеры

Лучший способ привыкнуть к формату — разобрать несколько готовых строк. Звёздочка с дробью `*/30` означает «каждые 30 единиц», поэтому такая запись запускает команду раз в полчаса:

```cron
*/30 * * * * command
```

Здесь минута зафиксирована на `0`, а час свободен — значит, команда срабатывает каждый час, ровно в начале часа:

```cron
0 * * * * command
```

Минута и час равны `0`, а последнее поле — день недели — равно `0` (воскресенье). Получается «в полночь каждое воскресенье»:

```cron
0 0 * * 0 command
```

А здесь задан день месяца `15` — команда запустится в полночь 15-го числа любого месяца:

```cron
0 0 15 * * command
```

Заданы и день месяца, и месяц — `1` и `1`. Это «в полночь 1 января», то есть раз в год:

```cron
0 0 1 1 * command
```

И отдельный случай — ключевое слово вместо пяти полей. `@reboot` запускает команду не по времени, а при каждой загрузке системы:

```cron
@reboot command
```

> **Типичные ошибки.**
>
> - **Команда работает в терминале, а в cron — нет.** У cron-задачи минимальная среда: переменная `PATH` короче, рабочий каталог — домашний, переменных вроде `JAVA_HOME` или `VIRTUAL_ENV` нет. Всегда указывайте абсолютные пути (`/usr/bin/python3` вместо `python3`), а нужные переменные задавайте прямо в строке crontab.
> - **`%` без экранирования.** В crontab символ `%` означает «перевод строки в `STDIN` команды». Запись `$(date +%F)` остановит парсер на первом `%`. В строке crontab `%` нужно экранировать: `\%F`. Это «классическая» ловушка, через которую проходят все.
> - **Вывод задачи не виден.** Если команда что-то печатает, cron по умолчанию шлёт это письмом локальному пользователю — а почтового агента в современных серверах часто нет, и письмо теряется. Чтобы видеть результат, перенаправляйте вывод в файл: `>>/var/log/mybackup.log 2>&1`.
> - **Перепутаны «день месяца» и «день недели».** Строка `0 12 1 * 1` — это «в полдень 1-го числа любого месяца, **и** по понедельникам», а не «в полдень понедельника 1-го числа». Cron комбинирует эти поля по «или», если оба заданы. Чтобы получить пересечение, оставляйте одно из полей звёздочкой.

## Практическое задание

1. Посмотрите текущий crontab через `crontab -l`.
2. Добавьте задачу, которая раз в минуту пишет дату в файл.
3. Проверьте результат через несколько минут.
4. Удалите тестовую задачу.
