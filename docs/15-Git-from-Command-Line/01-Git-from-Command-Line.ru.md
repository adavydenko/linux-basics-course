# Git из командной строки

Git нужен студентам для сдачи лабораторных, хранения кода, воспроизведения изменений и совместной работы. В этом курсе достаточно базового workflow из командной строки.

## Цели раздела

После раздела студент должен уметь:

- клонировать репозиторий;
- смотреть состояние рабочей директории;
- добавлять файлы в индекс;
- создавать коммиты;
- получать и отправлять изменения;
- использовать `.gitignore`.

## Базовый workflow

Клонировать репозиторий:

```shell
$ git clone https://example.com/course/lab.git
$ cd lab
```

Посмотреть состояние:

```shell
$ git status
```

Посмотреть изменения:

```shell
$ git diff
```

Добавить файл в индекс:

```shell
$ git add report.md
```

Создать коммит:

```shell
$ git commit -m "Add lab report"
```

Получить изменения:

```shell
$ git pull
```

Отправить изменения:

```shell
$ git push
```

## `.gitignore`

В репозиторий не нужно добавлять временные файлы, большие результаты вычислений, секреты и локальные окружения.

Пример `.gitignore`:

```gitignore
.venv/
__pycache__/
*.log
*.tmp
results/
.DS_Store
```

## Хороший коммит

Хороший коммит:

- относится к одной задаче;
- имеет понятное сообщение;
- не содержит случайных временных файлов;
- может быть проверен через `git diff`.

## Практическое задание

1. Создайте новый репозиторий.
2. Добавьте `README.md`.
3. Создайте `.gitignore`.
4. Сделайте первый коммит.
5. Измените `README.md`, посмотрите `git diff`.
6. Сделайте второй коммит.

3. Зачем нужен `.gitignore`?
4. Почему не стоит коммитить большие результаты вычислений?
