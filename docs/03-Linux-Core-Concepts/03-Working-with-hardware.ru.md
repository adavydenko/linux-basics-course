# Работа с оборудованием

- Перейти к [видеоуроку](https://kodekloud.com/topic/working-with-hardware/)

В этом разделе мы посмотрим, как Linux работает с аппаратными ресурсами системы и как получать информацию об устройствах из командной строки.

## Цели раздела

После раздела студент должен уметь:

- объяснить роль драйверов устройств;
- понимать, зачем нужен `udev`;
- читать сообщения ядра через `dmesg`;
- получать информацию о PCI-устройствах, блочных устройствах, CPU и памяти;
- понимать, когда команде нужны права суперпользователя.

## Как Linux видит устройства

Рассмотрим пример с USB-диском. Когда USB-устройство подключается к системе, драйвер устройства в пространстве ядра обнаруживает изменение состояния и создаёт событие. Такое событие называется `uevent`.

Событие отправляется в пространство пользователя диспетчеру устройств `udev`. Служба `udev` динамически создаёт узел устройства в файловой системе `/dev/`. После завершения процесса новый диск должен быть виден внутри `/dev/`.

## Сообщения ядра: `dmesg`

Команда `dmesg` показывает сообщения из кольцевого буфера ядра (ring buffer). Во время загрузки Linux ядро создаёт много сообщений: об обнаруженном оборудовании, драйверах и ошибках настройки.

```shell
$ dmesg
$ dmesg | grep -i usb
```

Эта команда полезна при диагностике подключённых устройств, особенно USB-дисков, сетевых адаптеров и проблем с драйверами.

## Информация от `udev`

`udevadm` — это утилита управления `udev`. Она позволяет запрашивать информацию из базы устройств:

```shell
$ udevadm info --query=path --name=/dev/sda5
```

Команда `udevadm monitor` слушает новые события ядра и выводит подробности: путь устройства и имя устройства. Это удобно, когда нужно понять, что происходит при подключении или отключении оборудования.

```shell
$ udevadm monitor
```

## Просмотр оборудования

Чтобы вывести PCI-устройства, настроенные в системе, используйте `lspci`. Примеры PCI-устройств: Ethernet-карты, RAID-контроллеры, видеокарты и беспроводные адаптеры.

```shell
$ lspci
```

![pci](../../images/pci.PNG)

Чтобы вывести информацию о блочных устройствах, используйте `lsblk`:

```shell
$ lsblk
```

![block-device](../../images/block-device.PNG)

Чтобы показать подробную информацию о CPU, включая архитектуру и режимы работы, используйте:

```shell
$ lscpu
```

Чтобы посмотреть доступную память:

```shell
$ lsmem --summary
```

Альтернативная команда для просмотра занятой и свободной памяти:

```shell
$ free -m
```

Чтобы получить подробную информацию обо всём оборудовании машины, используйте:

```shell
$ lshw
```

Некоторые команды требуют прав `root` или суперпользователя. В таких случаях используется `sudo`:

```shell
$ sudo lshw
```

## Практическое задание

1. Выполните `lsblk` и найдите блочные устройства типа `disk`.
2. Выполните `lscpu` и определите количество CPU или ядер.
3. Выполните `free -m` и запишите общий объём памяти.
4. Выполните `dmesg | grep -i usb` и объясните, есть ли в выводе сообщения об USB.
