## Ядро Linux

Если вы работали с любой операционной системой, вы уже сталкивались с понятием ядра.

Ядро Linux монолитное: оно самостоятельно выполняет планирование процессов, управление памятью и многие другие базовые операции. При этом ядро Linux модульное: его возможности можно расширять через динамически загружаемые модули ядра.

![linux-kernel](../../images/linux-kernel.PNG)

Для простой аналогии представьте библиотеку колледжа. Библиотекарь управляет доступом к книгам и помогает читателям получить нужные материалы. В этой аналогии библиотекарь похож на ядро Linux: он не делает работу пользователя вместо него, но управляет доступом к ресурсам.

![library](../../images/library.PNG)

Основные задачи ядра:

1. Управление памятью.
2. Управление процессами.
3. Работа с драйверами устройств.
4. Системные вызовы и безопасность.

## Версии ядра Linux

Чтобы получить информацию о ядре, используется команда `uname`. Без опций она показывает только, что система использует ядро Linux:

```shell
$ uname
```

Чтобы вывести версию ядра, используйте `uname -r`. Чтобы получить более подробную информацию, используйте `uname -a`:

```shell
$ uname -r
$ uname -a
```

![kernel-versions](../../images/kernel-versions.PNG)

## Пространство ядра и пространство пользователя

Одна из важных задач ядра — управление памятью. Память условно разделяется на две области.

1. Пространство ядра (kernel space):
   - код ядра;
   - расширения ядра;
   - драйверы устройств.
2. Пространство пользователя (user space):
   - пользовательские программы на C, Java, Python, Ruby и других языках;
   - контейнеры;
   - обычные процессы, запущенные пользователями.

![memory-management](../../images/memory-management.PNG)

Пользовательские программы работают с данными в памяти и на диске, но не обращаются к оборудованию напрямую. Для доступа к ресурсам они делают специальные запросы к ядру — системные вызовы (system calls).

Примеры системных вызовов: выделение памяти, открытие файла, чтение данных, запись данных.

![user-space](../../images/user-space.PNG)

Например, когда программа открывает файл `/etc/os-release`, чтобы посмотреть информацию об установленной системе, она обращается к ядру через системный вызов.

![user-space1](../../images/user-space1.PNG)

## Практическое задание

1. Выполните `uname`, `uname -r` и `uname -a`. Сравните вывод.
2. Найдите версию ядра и запишите её в отчёт.
3. Откройте файл `/etc/os-release` через `cat` и объясните, почему для этого нужен системный вызов.
4. Найдите в системе процессы пользователя через `ps`.
