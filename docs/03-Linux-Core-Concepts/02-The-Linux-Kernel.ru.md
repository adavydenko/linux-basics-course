# Ключевые понятия Linux

- [Доп. видео](https://kodekloud.com/topic/linux-kernel/)

Вы запускаете программу, она читает файл, выводит результат на экран — и за всем этим стоит один незаметный посредник, ядро. Прикладной программист его обычно не видит, но именно ядро решает, кто получит память, чей процесс выполнится следующим и можно ли вообще обратиться к диску. Чтобы понимать, что происходит «под капотом» Linux, начнём с него: что такое ядро, чем пространство ядра отличается от пространства пользователя и как программы вообще достукиваются до системных ресурсов.

## Цели раздела

После раздела студент должен уметь:

- отличать kernel space от user space;

## Ядро Linux

Ядро (kernel) — это центральная часть операционной системы, тот её слой, который напрямую общается с оборудованием. Если вы работали с Windows или macOS, вы уже косвенно имели дело с ядром, просто оно было скрыто за графической оболочкой.

У ядра Linux есть две важные характеристики, и на первый взгляд они кажутся противоречивыми. Оно **монолитное**: планирование процессов, управление памятью и другие базовые операции ядро выполняет само, в едином большом блоке кода. И одновременно оно **модульное**: его возможности можно расширять на ходу — подгружать и выгружать модули ядра, например драйвер нового устройства, не пересобирая систему. То есть основа неделима, но к ней можно докручивать части по мере надобности.

![linux-kernel](../images/ru_linux-kernel.svg)

Чтобы это уложилось в голове, представьте библиотеку колледжа. Библиотекарь не пишет за вас курсовую — но без него вы не доберётесь до нужных книг: он управляет доступом к фондам, выдаёт материалы, следит за порядком. Ядро Linux в этой аналогии — библиотекарь: оно не делает работу программ вместо них, но контролирует доступ ко всем ресурсам системы.

![library](../images/ru_library.svg)

Если свести роль ядра к списку, получится четыре основные задачи:

1. Управление памятью.
2. Управление процессами.
3. Работа с драйверами устройств.
4. Системные вызовы и безопасность.

## Версии ядра Linux

Ядро Linux постоянно развивается, и в разных системах работают разные его версии. Узнать, какая версия у вас, помогает команда `uname` (unix name). Запущенная без опций, она довольно скупа — сообщает лишь, что система использует ядро Linux:

```shell
$ uname
```

Гораздо полезнее опции. Флаг `-r` (release) выводит конкретную версию ядра, а `-a` (all) — всё, что `uname` вообще знает: версию, имя хоста, архитектуру и дату сборки:

```shell
$ uname -r
$ uname -a
```

Версию вида `4.15.0-72-generic` стоит уметь читать по частям — каждая цифра в ней несёт свой смысл:

| Часть | Что означает |
|---|---|
| `4` | основная версия ядра |
| `15` | старшая версия (major) |
| `0` | младшая версия (minor) |
| `72` | номер патч-релиза |
| `generic` | метка сборки дистрибутива (например, общее ядро Ubuntu) |

Эти цифры пригодятся, когда понадобится проверить совместимость драйвера или программы с ядром.

## Пространство ядра и пространство пользователя

Управляя памятью, ядро не сваливает весь код в общую кучу. Память условно делится на две области, и это разделение — основа стабильности и безопасности системы.

1. **Пространство ядра** (kernel space) — сюда нет доступа обычным программам. Здесь работают:
   - код ядра;
   - расширения ядра;
   - драйверы устройств.
2. **Пространство пользователя** (user space) — область для всего остального:
   - пользовательские программы на C, Java, Python, Ruby и других языках;
   - контейнеры;
   - обычные процессы, запущенные пользователями.

![memory-management](../images/ru_memory-management.svg)

Смысл такой изоляции прост: ошибка в пользовательской программе не должна обрушить всю систему. Поэтому программы из user space работают со своими данными в памяти и на диске, но напрямую к оборудованию не лезут. Когда им что-то нужно от системы, они вежливо просят об этом ядро — делают **системный вызов** (system call).

Системными вызовами оформляется почти всё, что выходит за пределы собственной памяти программы: выделить память, открыть файл, прочитать или записать данные.

![user-space](../images/ru_user-space.svg)

Вот конкретный пример. Программа хочет узнать, какая система установлена, и для этого открывает файл `/etc/os-release`. Сама она открыть файл не может — это работа с диском, ресурсом ядра, — поэтому за кулисами выполняется системный вызов: программа просит ядро, а ядро уже читает файл и отдаёт данные обратно.

![user-space1](../images/ru_user-space1.svg)

## Практическое задание

1. Выполните `uname`, `uname -r` и `uname -a`. Сравните вывод.
2. Найдите версию ядра и запишите её в отчёт.
3. Откройте файл `/etc/os-release` через `cat` и объясните, почему для этого нужен системный вызов.
4. Найдите в системе процессы пользователя через `ps`.
