## Основные типы файлов

В Linux многие объекты представлены как файлы. Даже каталог — это специальный тип файла.

Основные типы:

1. Обычные файлы.
2. Каталоги.
3. Специальные файлы.

![file-types](../../images/file-types.PNG "file-types")

Специальные файлы делятся на несколько видов:

1. Символьные устройства.
   - Представляют устройства в файловой системе.
   - Обычно находятся в `/dev`.
   - Примеры: клавиатура, мышь, терминал.
2. Блочные устройства.
   - Представляют устройства, которые работают с данными блоками.
   - Обычно находятся в `/dev`.
   - Примеры: диски и разделы.
3. Ссылки.
   - Позволяют связать одно или несколько имён с данными файла.
   - Бывают жёсткие ссылки (hard links) и символические ссылки (symbolic links).
4. Сокеты.
   - Специальные файлы для обмена данными между процессами.
5. Именованные каналы.
   - Специальные файлы, которые позволяют одному процессу передавать данные другому.

## Как определить тип файла

Первый способ — команда `file`:

```shell
$ file ~/
/home/user/: directory

$ file ./README.md
./README.md: ASCII text

$ file bash-script.sh
$ file insync1000.sock
$ file /home/michael/bash-script
```

Второй способ — команда `ls -ld`:

```shell
$ ls -ld ~/
drwxr-xr-x 109 user user 3488 Feb 15 22:59 /home/user/

$ ls -ld ./README.md
-rw-r--r-- 1 user user 5205 Feb 15 21:24 ./README.md
```

Тип файла можно определить по первому символу строки прав доступа. Например, в строке `drwxr-xr-x` первый символ `d`, значит это каталог.

| Тип файла | Первый символ |
| --- | --- |
| Каталог | `d` |
| Обычный файл | `-` |
| Символьное устройство | `c` |
| Символическая ссылка | `l` |
| Сокет | `s` |
| Именованный канал | `p` |
| Блочное устройство | `b` |

## Практическое задание

1. Выполните `file` для домашнего каталога, обычного текстового файла и `/dev/null`.
2. Выполните `ls -ld` для тех же объектов.
3. Найдите в `/dev` хотя бы одно блочное устройство и одно символьное устройство.
4. Объясните, чем результат `file` удобнее вывода `ls -ld`.
