# Типы файлов в Linux

- [Доп. видео](https://kodekloud.com/topic/file-types/)

В Linux есть негласное правило: почти всё в системе — это файл. Не только документы и фотографии, но и каталоги, диски, даже подключённая клавиатура представлены файлами. Звучит странно, но именно эта идея делает систему такой стройной — с любым объектом можно работать одними и теми же инструментами. Разберёмся, какие бывают типы файлов и как определить, с чем именно вы имеете дело.

## Цели раздела

После раздела студент должен уметь:

- объяснить принцип “всё является файлом”;
- назвать основные типы файлов Linux;
- определить тип файла через `file`;
- определить тип файла по первому символу в выводе `ls -l`;
- понимать, зачем нужны специальные файлы в `/dev`.

## Основные типы файлов

Раз файлом в Linux может быть что угодно, «файлы» внутри себя делятся на типы. На верхнем уровне их три:

1. Обычные файлы — то, что мы привыкли называть файлами: тексты, изображения, программы.
2. Каталоги — да, каталог тоже файл, просто особый: внутри него хранится список того, что в нём лежит.
3. Специальные файлы — самая интересная группа: за ними скрываются устройства, каналы связи и другие системные сущности.

![file-types](../images/ru_file-types.svg "file-types")

Третья группа неоднородна, поэтому специальные файлы делятся ещё на несколько видов:

1. Символьные устройства.
   - Представляют устройства в файловой системе.
   - Обычно находятся в `/dev`.
   - Примеры: клавиатура, мышь, терминал.
2. Блочные устройства.
   - Представляют устройства, которые работают с данными блоками.
   - Обычно находятся в `/dev`.
   - Примеры: диски и разделы.
3. Ссылки.
   - Позволяют связать одно или несколько имён с данными файла.
   - Бывают жёсткие ссылки (hard links) и символические ссылки (symbolic links).
4. Сокеты.
   - Специальные файлы для обмена данными между процессами.
5. Именованные каналы.
   - Специальные файлы, которые позволяют одному процессу передавать данные другому.

## Как определить тип файла

На практике типов так много, что угадывать не стоит — лучше спросить систему. Есть два удобных способа.

Первый — команда `file`. Она заглядывает в содержимое объекта и отвечает по-человечески, словами:

```shell
$ file ~/
/home/user/: directory

$ file ./README.md
./README.md: ASCII text

$ file bash-script.sh
$ file insync1000.sock
$ file /home/michael/bash-script
```

Обратите внимание на ответы: для домашнего каталога `file` пишет `directory`, для `README.md` — `ASCII text`. То есть она не просто различает «файл и каталог», но и подсказывает, что внутри обычного файла.

Второй способ — уже знакомая `ls` с флагами `-ld`. Здесь тип придётся прочитать самому:

```shell
$ ls -ld ~/
drwxr-xr-x 109 user user 3488 Feb 15 22:59 /home/user/

$ ls -ld ./README.md
-rw-r--r-- 1 user user 5205 Feb 15 21:24 ./README.md
```

Секрет — в самом первом символе строки прав доступа. В выводе для каталога строка начинается с `drwxr-xr-x`: первая буква `d` означает «directory», каталог. А строка `-rw-r--r--` для `README.md` начинается с дефиса — это обычный файл. Полное соответствие первых символов и типов собрано в таблице ниже.

| Тип файла | Первый символ |
| --- | --- |
| Каталог | `d` |
| Обычный файл | `-` |
| Символьное устройство | `c` |
| Символическая ссылка | `l` |
| Сокет | `s` |
| Именованный канал | `p` |
| Блочное устройство | `b` |

## Разобранный пример: «что это вообще лежит в `/dev/sda1`?»

**Задача.** Студент видит в каталоге `/dev` много объектов с непонятными именами — `sda`, `sda1`, `tty1`, `null`, `random` — и хочет понять, какие из них устройства, какие нет, и какого они типа.

**Решение.**

1. Берём один объект и спрашиваем у системы напрямую: `file /dev/sda1`. В ответ получаем «block special» — это блочное устройство (диск или раздел).
2. Тот же объект через `ls -ld /dev/sda1` начнётся с буквы `b` — `brw-rw----`. Первая буква и есть тип; здесь `b` подтверждает блочное устройство.
3. Сравниваем с `/dev/tty1`: `ls -ld /dev/tty1` покажет первую букву `c` — символьное устройство, к нему обращаются побайтно (виртуальный терминал).
4. Заглянем в `/dev/null` — он начинается с `c` тоже: это символьное устройство «чёрная дыра», в которое можно писать что угодно.
5. И наконец `/etc/hostname` — `ls -ld /etc/hostname` начинается с `-`, это обычный файл с текстом внутри. Команда `file /etc/hostname` подтвердит: «ASCII text».

**Проверка.** В каталоге `/dev` выберите три объекта наугад и для каждого назовите тип по первой букве `ls -ld`. Затем проверьте себя командой `file` — оба способа должны давать совпадающие ответы (хоть и разными словами).

## Практическое задание

1. Выполните `file` для домашнего каталога, обычного текстового файла и `/dev/null`.
2. Выполните `ls -ld` для тех же объектов.
3. Найдите в `/dev` хотя бы одно блочное устройство и одно символьное устройство.
4. Объясните, чем результат `file` удобнее вывода `ls -ld`.
