# Иерархия файловой системы

- Перейти к [видеоуроку](https://kodekloud.com/topic/filesystem-hierarchy/)

В этом разделе разберём иерархию файловой системы Linux.

## Цели раздела

После раздела студент должен уметь:

- объяснить, что файловая система Linux имеет единый корень `/`;
- назвать назначение основных системных каталогов;
- понимать разницу между `/home`, `/root`, `/etc`, `/var`, `/usr`, `/dev`;
- смотреть примонтированные файловые системы через `df`.

## Единый корень

Linux использует древовидную файловую систему с одним корнем `/`. Все файлы, каталоги, устройства и точки монтирования находятся внутри этого дерева.

Основные каталоги:

- `/home` — домашние каталоги обычных пользователей. Домашний каталог пользователя `root` обычно находится отдельно: `/root`.
- `/opt` — место для установки стороннего программного обеспечения.
- `/mnt` — каталог для временного монтирования файловых систем.
- `/tmp` — временные данные.
- `/media` — часто используется для внешних носителей.
- `/dev` — специальные файлы устройств: блочные и символьные.
- `/bin` — базовые исполняемые файлы, например `cp`, `mv`, `mkdir`.
- `/etc` — конфигурационные файлы системы.
- `/lib` и `/lib64` — общие библиотеки, используемые программами.
- `/usr` — пользовательские приложения, библиотеки и данные, установленные системой.
- `/var` — изменяемые данные: логи, очереди, кэш, почтовые файлы.

## Просмотр файловых систем

Чтобы вывести примонтированные файловые системы, выполните `df`. Часто используется вариант `df -hP`, где `-h` делает размеры удобными для чтения, а `-P` включает переносимый формат вывода.

```shell
$ df -hP
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        40G   12G   26G  32% /
tmpfs           2.0G     0  2.0G   0% /dev/shm
/dev/sdb1       100G   45G   55G  45% /data
```

Важная колонка — `Mounted on`: она показывает, в какую точку дерева подключена файловая система.

## Практическое задание

1. Выполните `ls /` и найдите основные системные каталоги.
2. Выполните `df -hP` и определите, какая файловая система смонтирована в `/`.
3. Посмотрите содержимое `/etc`, `/var`, `/usr`, `/dev`.
4. Объясните, почему пользовательские файлы лучше хранить в `/home`, а не в `/tmp`.
