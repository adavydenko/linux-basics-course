# Иерархия файловой системы

- [Доп. видео](https://kodekloud.com/topic/filesystem-hierarchy/)

В Windows вы привыкли к дискам `C:`, `D:`, каждый со своим деревом. В Linux всё иначе: дерево одно на всю систему, и у него единственный корень. Сначала это сбивает с толку, но у такого устройства есть железная логика — и, разобравшись в ней, вы перестанете теряться в чужой системе. Пройдёмся по основным каталогам и узнаем, что в каждом из них хранится.

## Цели раздела

После раздела студент должен уметь:

- объяснить, что файловая система Linux имеет единый корень `/`;
- назвать назначение основных системных каталогов;
- понимать разницу между `/home`, `/root`, `/etc`, `/var`, `/usr`, `/dev`;
- смотреть примонтированные файловые системы через `df`.

## Единый корень

Файловая система Linux — это дерево с единственным корнем, который обозначается одним символом `/`. Всё без исключения — файлы, каталоги, устройства, подключённые диски — находится где-то внутри этого дерева. Отдельных «дисков с буквами» нет: новый накопитель не создаёт новое дерево, а подключается веткой к уже существующему.

Каждый каталог верхнего уровня отвечает за свою категорию данных — это не свалка, а продуманное распределение. Вот основные из них:

- `/home` — домашние каталоги обычных пользователей. Домашний каталог пользователя `root` обычно находится отдельно: `/root`.
- `/opt` — место для установки стороннего программного обеспечения.
- `/mnt` — каталог для временного монтирования файловых систем.
- `/tmp` — временные данные.
- `/media` — часто используется для внешних носителей.
- `/dev` — специальные файлы устройств: блочные и символьные.
- `/bin` — базовые исполняемые файлы, например `cp`, `mv`, `mkdir`.
- `/etc` — конфигурационные файлы системы.
- `/lib` и `/lib64` — общие библиотеки, используемые программами.
- `/usr` — пользовательские приложения, библиотеки и данные, установленные системой.
- `/var` — изменяемые данные: логи, очереди, кэш, почтовые файлы.

## Просмотр файловых систем

Раз диски подключаются ветками к общему дереву, возникает вопрос: как увидеть, что и куда подключено? Это показывает команда `df` (disk free). Чаще всего её запускают с флагами `-hP`: `-h` (human-readable) переводит размеры в привычные гигабайты вместо длинных чисел, а `-P` (portable) даёт стабильный, переносимый формат вывода.

```shell
$ df -hP
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        40G   12G   26G  32% /
tmpfs           2.0G     0  2.0G   0% /dev/shm
/dev/sdb1       100G   45G   55G  45% /data
```

Разберём вывод по колонкам. `Filesystem` — что подключено (обычно раздел диска); `Size`, `Used`, `Avail` — общий размер, занято и свободно; `Use%` — заполненность в процентах. Но самая показательная колонка здесь — `Mounted on`: она говорит, в какую точку общего дерева подключена файловая система. В примере выше раздел `/dev/sda1` смонтирован в корень `/`, а отдельный диск `/dev/sdb1` — в каталог `/data`. Именно так несколько физических дисков и складываются в одно дерево.

## Практическое задание

1. Выполните `ls /` и найдите основные системные каталоги.
2. Выполните `df -hP` и определите, какая файловая система смонтирована в `/`.
3. Посмотрите содержимое `/etc`, `/var`, `/usr`, `/dev`.
4. Объясните, почему пользовательские файлы лучше хранить в `/home`, а не в `/tmp`.
