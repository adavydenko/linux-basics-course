# Последовательность загрузки Linux

- Перейти к [видеоуроку](https://kodekloud.com/topic/linux-boot-sequence/)

В этом разделе мы рассмотрим процесс загрузки Linux в упрощённом виде, разделив его на четыре основных этапа.

## Цели раздела

После раздела студент должен уметь:

- перечислить основные этапы загрузки Linux;
- объяснить роль BIOS/POST, загрузчика, ядра и init-процесса;
- понимать, где в современной системе появляется `systemd`;
- проверить, какой init используется в системе.

## Этапы загрузки

Процесс загрузки можно разбить на четыре этапа:

1. BIOS POST.
2. Загрузчик, например GRUB2.
3. Инициализация ядра.
4. Init-процесс.

![boot-sequence](../../images/boot-sequence.PNG)

Запустить процесс загрузки можно двумя способами:

- включить выключенное устройство;
- перезагрузить уже работающую систему.

## BIOS POST

Первый этап называется BIOS POST и почти не связан с самим Linux.

`POST` расшифровывается как `Power On Self Test`. На этом этапе BIOS проверяет, что основные аппаратные компоненты работают корректно. Если POST завершается ошибкой, компьютер может не перейти к следующему этапу загрузки.

![BIOS](../../images/BIOS.PNG)

## Загрузчик

После успешного POST BIOS загружает и выполняет код загрузчика с загрузочного устройства. В Linux файлы загрузчика обычно связаны с файловой системой `/boot`.

Загрузчик показывает экран выбора, где может быть несколько вариантов: например, разные версии ядра или разные операционные системы в dual boot.

После выбора загрузчик помещает ядро в память, передаёт ему параметры и отдаёт управление ядру.

Один из самых распространённых загрузчиков — `GRUB2` (`GRand Unified Bootloader Version 2`).

![boot-loader](../../images/boot-loader.PNG)

## Инициализация ядра

После загрузки выбранное ядро обычно распаковывается и начинает работу в памяти. На этом этапе ядро инициализирует оборудование, управляет памятью и готовит базовую среду для запуска пользовательского пространства.

Когда ядро готово, оно ищет init-процесс, который должен настроить user space и запустить необходимые системные службы.

![kernel-initialize](../../images/kernel-initialize.PNG)

## Init-процесс

В большинстве современных дистрибутивов Linux init-функцию выполняет демон `systemd`.

`systemd` приводит систему в рабочее состояние: монтирует файловые системы, запускает и управляет системными службами.

Сегодня `systemd` является распространённым стандартом, но раньше часто использовался другой init-процесс — `System V init` или `SysV init`. Например, он применялся в старых версиях RHEL 6 и CentOS 6.

Одно из преимуществ `systemd` по сравнению с `System V init` — сокращение времени загрузки за счёт параллельного запуска служб.

Чтобы проверить, какая init-система используется, выполните:

```shell
$ ls -l /sbin/init
```

Если используется `systemd`, вы увидите ссылку на `/lib/systemd/systemd`.

## Практическое задание

1. Выполните `ls -l /sbin/init`.
2. Определите, используется ли `systemd`.
3. Найдите каталог `/boot` и посмотрите его содержимое.
4. Объясните, почему загрузчик должен запустить ядро до появления пользовательских программ.
