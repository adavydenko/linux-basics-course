# Лабораторная работа: ядро Linux

- Перейти к [лабораторной работе](https://kodekloud.com/topic/lab-linux-kernel-lsmod-insmod-modprobe-uname/)

## Цели лабораторной работы

После выполнения работы студент должен уметь:

- определять версию ядра;
- читать сообщения ядра;
- находить блочные устройства;
- получать сведения о CPU и памяти.

## Задания

1. Проверить точную версию ядра, запущенного в системе:

   ```shell
   $ uname -r
   ```

2. Для версии `4.15.0-88-generic` определить номер версии ядра:

   ```text
   Первая часть версии: 4
   ```

3. Для версии `4.15.0-88-generic` определить major/minor-компоненты:

   ```text
   Major: 4
   Minor: 15
   ```

4. Вывести сообщения, сгенерированные ядром:

   ```shell
   $ dmesg
   ```

5. Посмотреть и посчитать блочные устройства типа `disk`:

   ```shell
   $ lsblk
   ```

6. Проверить количество физических ядер в системе:

   ```shell
   $ lscpu
   ```

   В выводе найдите `Core(s) per socket` и `Socket(s)`, затем перемножьте значения.

7. Проверить общий объём доступной памяти:

   ```shell
   $ lsmem --summary
   ```

## Что нужно сдать

- Вывод `uname -r`.
- Краткий ответ по структуре версии ядра.
- Вывод или фрагмент вывода `lsblk`, `lscpu`, `lsmem --summary`.
- Объяснение, зачем нужна команда `dmesg`.

## Контрольные вопросы

1. Какая команда показывает версию ядра?
2. Где можно увидеть сообщения ядра?
3. Какие устройства показывает `lsblk`?
4. Почему сведения о CPU и памяти важны для вычислительных задач?
