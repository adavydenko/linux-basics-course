# Лабораторная работа: ядро Linux

- [Доп. лабораторная](https://kodekloud.com/topic/lab-linux-kernel-lsmod-insmod-modprobe-uname/)

## Цели лабораторной работы

После выполнения работы студент должен уметь:

- определять версию ядра;
- читать сообщения ядра;
- находить блочные устройства;
- получать сведения о CPU и памяти.

## Задания

Эта работа закрепляет навык расспрашивать систему о её ядре и оборудовании. Сначала вы научитесь читать версию ядра, затем — снимать сведения о памяти, дисках и процессоре теми же `ls`-командами, что разбирались в разделах модуля.

1. Узнать точную версию ядра, которое сейчас запущено в системе:

   ```shell
   $ uname -r
   ```

2. Разобрать версию `4.15.0-88-generic` по частям и назвать номер версии ядра — это самое первое число:

   ```text
   Первая часть версии: 4
   ```

3. Для той же версии `4.15.0-88-generic` выделить основную (major) и дополнительную (minor) компоненты — первые два числа:

   ```text
   Major: 4
   Minor: 15
   ```

4. Вывести сообщения, которые сгенерировало ядро, — это поможет увидеть, что оно обнаружило при загрузке:

   ```shell
   $ dmesg
   ```

5. Посмотреть блочные устройства и посчитать среди них устройства типа `disk` (в выводе `lsblk` тип указан в колонке `TYPE`):

   ```shell
   $ lsblk
   ```

6. Определить количество физических ядер процессора:

   ```shell
   $ lscpu
   ```

   В выводе найдите строки `Core(s) per socket` и `Socket(s)` и перемножьте их значения — так получится общее число физических ядер.

7. Узнать общий объём доступной памяти:

   ```shell
   $ lsmem --summary
   ```

## Что нужно сдать

Оформите отчёт по [единому формату](../00-Lab-Environment/01-Lab-Environment-Setup.ru.md#формат-отчёта-по-лабораторной-работе). Дополнительно для этой работы приложите:

- Вывод `uname -r`.
- Краткий ответ по структуре версии ядра.
- Вывод или фрагмент вывода `lsblk`, `lscpu`, `lsmem --summary`.
- Объяснение, зачем нужна команда `dmesg`.
