# Лабораторная работа: уровни запуска и файловая система

- [Доп. лабораторная](https://kodekloud.com/topic/lab-linux-kernel-modules-boot-and-filetypes/)

## Цели лабораторной работы

После выполнения работы студент должен уметь:

- запускать команды с `sudo`;
- проверять init-систему;
- смотреть и менять target systemd по умолчанию;
- определять тип файлов;
- ориентироваться в назначении системных каталогов.

## Задания

Эта работа сводит воедино темы всего модуля: права суперпользователя, init-систему и targets, типы файлов и назначение системных каталогов. Часть заданий — практические команды, часть — короткие вопросы на понимание; и то и другое включите в отчёт.

1. Запустить команду, которой нужны права суперпользователя. Каталог `/root` принадлежит администратору, поэтому без `sudo` система откажет в доступе:

   ```shell
   $ sudo ls /root
   ```

2. Определить, какая init-система используется — `systemd` или SysV. Подсказку даёт ссылка `/sbin/init`: куда она ведёт, та система и работает:

   ```shell
   $ sudo ls -l /sbin/init
   ```

3. Узнать, в какой target systemd система загружается по умолчанию:

   ```shell
   $ sudo systemctl get-default
   ```

4. Переключить target systemd по умолчанию на неграфический режим `multi-user.target`:

   ```shell
   $ sudo systemctl set-default multi-user.target
   ```

5. Определить тип файла `/root/firefox.deb` командой `file`:

   ```shell
   $ sudo file /root/firefox.deb
   ```

6. Тем же способом определить тип файла `/root/sample_script.sh` и сравнить ответ с предыдущим:

   ```shell
   $ sudo file /root/sample_script.sh
   ```

7. Ответить на вопрос: в какой каталог принято устанавливать стороннюю IDE? Опирайтесь на назначение системных каталогов из раздела про иерархию:

   ```text
   Стороннее программное обеспечение обычно устанавливают в /opt.
   ```

8. Ответить, где система держит файлы блочных устройств, которые показывает `lsblk`:

   ```text
   Файлы устройств находятся в каталоге /dev.
   ```

9. Найти производителя сетевого контроллера (Ethernet Controller) в системе:

   ```shell
   $ sudo lshw
   ```

   В подробном выводе `lshw` отыщите сетевой раздел и поле `vendor` — в нём указан производитель.

## Что нужно сдать

Оформите отчёт по [единому формату](../00-Lab-Environment/01-Lab-Environment-Setup.ru.md#формат-отчёта-по-лабораторной-работе). Дополнительно для этой работы приложите:

- Вывод `systemctl get-default`.
- Вывод `ls -l /sbin/init`.
- Результаты `file` для двух файлов.
- Ответы по каталогам `/opt` и `/dev`.
- Фрагмент `lshw` с производителем сетевого контроллера.
