# Диски и разделы

- [Доп. видео](https://kodekloud.com/topic/disk-partitions/)

Один физический диск редко используют как единое целое — обычно его делят на разделы: один под систему, другой под данные, третий под подкачку. Прежде чем создавать файловые системы и что-либо монтировать, нужно понять, как Linux вообще «видит» диски и как устроена их разметка. С этого и начнём — а файловые системы, внешние хранилища и LVM подробно рассмотрим в следующих разделах.

## Цели раздела

После раздела студент должен уметь:

- выводить блочные устройства;
- понимать major/minor numbers;
- смотреть таблицу разделов;
- различать primary, extended и logical partitions;
- понимать базовый сценарий создания раздела через `gdisk`.

В этом разделе встретятся две схемы разметки. Историческая — MBR (Master Boot Record, главная загрузочная запись) — родилась вместе с IBM PC и до сих пор живёт в учебниках; её ограничения (не больше четырёх «полноценных» разделов и таблица только на дисках до 2 ТБ) мы разберём ниже. Современная — GPT (GUID Partition Table, таблица разделов на основе GUID) — пришла ей на смену: она не ограничена четырьмя разделами и поддерживает диски в петабайты. Команды, с которыми вы будете работать, тоже разные: `fdisk` исторически для MBR, `gdisk` — её GPT-аналог.

## Блочные устройства

В Linux есть принцип: «всё — файл». Диски не исключение. Каждый накопитель представлен в системе специальным файлом — блочным устройством. Так называют файлы, за которыми стоят устройства хранения: жёсткие диски, SSD, USB-накопители и их разделы. «Блочные» — потому что данные на них читаются и пишутся блоками фиксированного размера, а не побайтно.

Самый наглядный способ увидеть все накопители — команда `lsblk` (list block devices):

```shell
$ lsblk
```

Она показывает диски и их разделы деревом — сразу видно, что во что вложено. Сами файлы блочных устройств живут в каталоге `/dev`; отличить их можно по букве `b` в начале строки `ls -l`:

```shell
$ ls -l /dev/ | grep "^b"
```

У каждого такого файла есть пара чисел — major и minor number. Major number указывает на тип устройства (его драйвер): например, `8` обычно соответствует SCSI/SATA-дискам с именами вида `sd`. Современные NVMe-накопители используют другой драйвер, и у них major будет `259` (а имена вроде `nvme0n1`) — поэтому жёсткой пары «major=8 — это диск» нет. Minor number различает между собой конкретные устройства и разделы одного типа.

Чтобы заглянуть в разметку конкретного диска и увидеть его разделы, используют `fdisk` с опцией `-l`:

```shell
$ sudo fdisk -l /dev/sda
```

## Типы разделов

![Partition](../images/ru_partition.svg)

Историческая схема разметки (MBR) не позволяет сделать сколько угодно полноценных разделов, поэтому в ней есть три их разновидности:

- `PRIMARY` — основной раздел; именно с него может загружаться операционная система.
- `EXTENDED` — расширенный раздел. Сам по себе данные он не хранит, его задача — служить «контейнером» для логических разделов, чтобы обойти ограничение на число основных.
- `LOGICAL` — логический раздел, который размещается внутри расширенного.

## Создание разделов

Современная схема разметки — GPT, и для работы с ней удобен `gdisk` (GPT fdisk). Он похож на классический `fdisk`, но рассчитан именно на GPT. Запускается интерактивно, на нужном диске:

```shell
$ sudo gdisk /dev/sdb
```

Дальше `gdisk` ждёт от вас односимвольных команд:

- `?` — показать справку;
- `n` — создать новый раздел (new);
- `p` — вывести текущую таблицу разделов (print);
- `w` — записать изменения на диск (write);
- `q` — выйти без сохранения (quit).

Ключевая идея: пока вы не дали команду `w`, на диске ничего не меняется — все правки живут только в памяти. Так выглядит типичный сценарий создания одного раздела:

```text
Command (? for help): n
Partition number (1-128, default 1): 1
First sector (..., default = 2048): 2048
Last sector (...): 41943006
Hex code or GUID (L to show codes, Enter = 8300):
Command (? for help): w
Do you want to proceed? (Y/N): Y
```

`gdisk` спрашивает номер раздела, его первый и последний секторы (то есть начало и размер) и тип. На большинстве вопросов достаточно нажать Enter и согласиться со значением по умолчанию. После записи стоит убедиться, что раздел действительно появился:

```shell
$ sudo fdisk -l /dev/sdb
```

## Важное предупреждение

Команды разметки диска могут уничтожить данные. Их нужно выполнять только на учебном диске или в виртуальной машине, где потеря данных безопасна.
