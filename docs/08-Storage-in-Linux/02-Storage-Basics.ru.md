# Диски и разделы

- Перейти к [видеоуроку](https://kodekloud.com/topic/disk-partitions/)

В этом разделе мы разберём разделы дисков, файловые системы, внешние хранилища и LVM.

## Цели раздела

После раздела студент должен уметь:

- выводить блочные устройства;
- понимать major/minor numbers;
- смотреть таблицу разделов;
- различать primary, extended и logical partitions;
- понимать базовый сценарий создания раздела через `gdisk`.

![Disk](../../images//disk.PNG)

## Блочные устройства

Блочные устройства — это специальные файлы, которые представляют устройства хранения: жёсткие диски, SSD, USB-накопители и разделы.

Вывести блочные устройства:

```shell
$ lsblk
```

Показать блочные файлы в `/dev`:

```shell
$ ls -l /dev/ | grep "^b"
```

Major number указывает тип устройства. Например, значение `8` часто соответствует SCSI/SATA-дискам, имена которых начинаются с `sd`. Minor number различает отдельные физические или логические устройства.

Посмотреть таблицу разделов:

```shell
$ sudo fdisk -l /dev/sda
```

## Типы разделов

![Part](../../images//partition.PNG)

- `PRIMARY` — основной раздел, может использоваться для загрузки ОС.
- `EXTENDED` — расширенный раздел; сам по себе не используется для данных, но может содержать logical partitions.
- `LOGICAL` — логический раздел внутри extended partition.

## Создание разделов

`gdisk` — инструмент для работы с GPT-разметкой. Он похож на `fdisk`, но лучше подходит для GPT.

Создать раздел на `/dev/sdb`:

```shell
$ sudo gdisk /dev/sdb
```

Внутри `gdisk` можно использовать:

- `?` — показать справку;
- `n` — создать новый раздел;
- `p` — вывести таблицу разделов;
- `w` — записать изменения на диск;
- `q` — выйти без сохранения.

Пример сценария:

```text
Command (? for help): n
Partition number (1-128, default 1): 1
First sector (..., default = 2048): 2048
Last sector (...): 41943006
Hex code or GUID (L to show codes, Enter = 8300):
Command (? for help): w
Do you want to proceed? (Y/N): Y
```

После создания раздела проверьте результат:

```shell
$ sudo fdisk -l /dev/sdb
```

## Важное предупреждение

Команды разметки диска могут уничтожить данные. Их нужно выполнять только на учебном диске или в виртуальной машине, где потеря данных безопасна.
