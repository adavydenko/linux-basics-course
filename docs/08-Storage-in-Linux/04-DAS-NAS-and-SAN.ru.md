# DAS, NAS и SAN

- Перейти к [видеоуроку](https://kodekloud.com/topic/das-nas-and-san/)

Теперь, когда базовые понятия хранилищ в Linux знакомы, рассмотрим внешние хранилища.

## Цели раздела

После раздела студент должен уметь:

- различать DAS, NAS и SAN;
- понимать, как NFS предоставляет каталоги по сети;
- знать назначение `/etc/exports`;
- экспортировать NFS-каталоги.

## Виды внешнего хранилища

- `DAS` (`Direct Attached Storage`) — хранилище подключено напрямую к хосту, которому нужно место.
- `NAS` (`Network Attached Storage`) — сетевое файловое хранилище, похожее по идее на NFS-сервер.
- `SAN` (`Storage Area Network`) — технология высокоскоростного блочного хранилища, часто через Fibre Channel.

## NFS

NFS хранит и отдаёт данные как файлы, а не как блоки. Он работает по модели клиент-сервер.

NFS-сервер хранит конфигурацию экспорта в `/etc/exports`. Этот файл определяет, какие клиенты могут получать доступ к каталогам сервера.

Пример `/etc/exports`:

```text
/software/repos 10.61.35.201 10.61.35.202 10.61.35.203
```

Экспортировать все монтирования, описанные в `/etc/exports`:

```shell
$ sudo exportfs -a
```

Экспортировать каталог вручную:

```shell
$ sudo exportfs -o rw 10.61.35.201:/software/repos
```
