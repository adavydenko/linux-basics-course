# Оболочка Bash

## Цели раздела

После раздела студент должен уметь:

- перечислить популярные оболочки Linux;
- узнать текущую оболочку пользователя;

- создавать alias для часто используемых команд;

- понимать роль переменной `PATH`;
- настраивать приглашение Bash через `PS1`.

## Виды оболочек

В Linux существует несколько видов shell. Самые известные:

- Bourne Shell (`sh`);
- C Shell (`csh` или `tcsh`);
- Korn Shell (`ksh`);
- Z Shell (`zsh`);
- Bourne Again Shell (`bash`).

Чтобы проверить оболочку, заданную для текущего пользователя, выполните:

```shell
$ echo $SHELL
```

Чтобы изменить оболочку по умолчанию, используется команда `chsh`. Она запросит пароль, а затем имя новой оболочки. Изменение обычно становится заметно после входа в новую терминальную сессию.

```shell
$ chsh
```

## Возможности Bash

### Автодополнение команд

Bash поддерживает автодополнение. Если набрать часть команды, пути или имени файла и нажать `Tab`, оболочка попробует дополнить ввод.

![bash-auto](../../images/bash-auto.PNG)

![bash-auto1](../../images/bash-auto1.PNG)

Автодополнение ускоряет работу и уменьшает количество опечаток.

### Alias

Alias — это короткое имя для команды или набора команд. Например, можно создать alias `dt` для команды `date`:

```shell
$ date
$ alias dt=date
$ dt
```

Alias полезен для часто повторяющихся команд, но его нужно использовать аккуратно: слишком много нестандартных сокращений может мешать чтению чужих инструкций.

### История команд

Команда `history` выводит список ранее выполненных команд:

```shell
$ history
```

История помогает повторять команды, искать прошлые действия и разбирать ход лабораторной работы.

## Переменные окружения Bash

Переменные окружения хранят настройки, которые shell и запущенные программы могут использовать во время работы.

Чтобы вывести значение переменной `SHELL`, выполните:

```shell
$ echo $SHELL
```

Чтобы посмотреть список всех переменных окружения, выполните:

```shell
$ env
```

Чтобы задать переменную только в текущей оболочке, можно написать:

```shell
$ OFFICE=caleston
```

Такое значение не будет автоматически передано дочерним процессам.

Чтобы переменная передавалась дочерним процессам, используйте `export`:

```shell
$ export OFFICE=caleston
```

Чтобы сохранять переменную после повторного входа в систему или перезагрузки, её можно добавить в файл `~/.profile` или `~/.pam_environment` в домашнем каталоге пользователя:

```shell
$ echo "export OFFICE=caleston" >> ~/.profile
$ echo "export OFFICE=caleston" >> ~/.pam_environment
```

Чтобы проверить значение переменной `LOGNAME`, выполните:

```shell
$ echo $LOGNAME
```

## Переменная `PATH`

Когда пользователь запускает внешнюю команду, shell ищет соответствующий исполняемый файл в каталогах, перечисленных в переменной `PATH`.

Чтобы увидеть эти каталоги, выполните:

```shell
$ echo $PATH
```

Чтобы проверить, где находится исполняемый файл команды, используйте `which`.

Синтаксис:

```shell
$ which <command>
```

Пример:

```shell
$ which obs-studio
```

Если программа установлена в нестандартный каталог, его можно добавить в `PATH`:

```shell
$ export PATH=$PATH:/opt/obs/bin
$ which obs-studio
```

Такое изменение действует только в текущей сессии, если не добавить его в профильный файл пользователя.

## Настройка приглашения Bash

После входа в shell пользователь видит приглашение Bash.

![bash-prompt](../../images/bash-prompt.PNG)

Его можно настроить так, чтобы оно показывало имя пользователя и имя хоста:

![bash-prompt1](../../images/bash-prompt1.PNG)

Приглашение Bash управляется специальными переменными окружения. Самая распространённая из них — `PS1`.

![bash-prompt2](../../images/bash-prompt2.PNG)

Чтобы посмотреть текущее значение `PS1`, выполните:

```shell
$ echo $PS1
```

Чтобы временно изменить приглашение и показывать только слово `ubuntu-server`, выполните:

```shell
$ PS1="ubuntu-server"
$ echo $PS1
```

Для более гибкой настройки используются специальные последовательности:

![bash-prompt3](../../images/bash-prompt3.PNG)

Например, так можно показать дату, время, имя текущего пользователя, имя хоста и текущий рабочий каталог:

```shell
$ PS1="[\d \t \u@\h:\w ] $ "
```
