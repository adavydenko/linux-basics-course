# Оболочка Bash

- [Доп. видео](https://kodekloud.com/topic/bash-shell/)

## Цели раздела

После раздела студент должен уметь:

- перечислить популярные оболочки Linux;
- узнать текущую оболочку пользователя;
- пользоваться автодополнением и историей команд;
- создавать alias для часто используемых команд;
- читать и задавать переменные окружения;
- понимать роль переменной `PATH`;
- настраивать приглашение Bash через `PS1`.

До сих пор мы говорили об «оболочке» вообще, как будто она одна. На самом деле оболочек несколько, и они различаются синтаксисом и возможностями. В этом разделе мы сосредоточимся на самой распространённой из них — Bash — и разберём, что именно делает её удобной в повседневной работе.

## Виды оболочек

Оболочка — это программа, и таких программ в мире Linux исторически появилось несколько. Самые известные:

- Bourne Shell (`sh`);
- C Shell (`csh` или `tcsh`);
- Korn Shell (`ksh`);
- Z Shell (`zsh`);
- Bourne Again Shell (`bash`).

Большинство современных дистрибутивов по умолчанию дают пользователю именно Bash, но проверить это стоит явно. Какая оболочка назначена текущему пользователю, показывает переменная `SHELL`:

```shell
$ echo $SHELL
```

В ответ вы увидите путь вроде `/bin/bash` — это и есть назначенная вам оболочка. Сменить её можно командой `chsh` (change shell): она спросит пароль, затем имя новой оболочки. Для смены **своей** оболочки `sudo` не нужен — `chsh` спрашивает ваш собственный пароль. А вот сменить оболочку другому пользователю — это уже административная задача (`sudo chsh -s /bin/zsh имя_пользователя`). Учтите, что назначение применится не сразу — оно вступит в силу со следующей терминальной сессии.

```shell
$ chsh
```

## Возможности Bash

Bash популярен не из-за названия, а из-за того, что он берёт на себя рутину. Три его возможности вы будете использовать буквально каждый день — автодополнение, псевдонимы и историю команд.

### Автодополнение команд

Набирать длинные имена файлов и путей целиком утомительно и легко ошибиться. Bash избавляет от этого: наберите начало команды, пути или имени файла и нажмите `Tab` — оболочка попробует дополнить ввод за вас. Если вариант один, Bash допишет его сразу; если их несколько — покажет список, и вам останется набрать ещё пару символов и снова нажать `Tab`. В итоге работа идёт быстрее, а опечаток в путях становится меньше.

### Alias

Некоторые команды вы набираете десятки раз в день, причём с одними и теми же флагами. Чтобы не повторять их целиком, Bash позволяет завести **alias** — короткое собственное имя для команды или цепочки команд. Например, заведём alias `dt` для `date`:

```shell
$ date
$ alias dt=date
$ dt
```

Теперь `dt` работает как `date`. Удобно — но есть нюанс: alias существует только в вашей сессии и в чужих инструкциях не встречается. Поэтому делайте псевдонимы для личного удобства, а в учебных и рабочих инструкциях пишите команды полностью, чтобы их мог повторить любой.

### История команд

Bash запоминает, что вы вводили. Команда `history` выводит список ранее выполненных команд:

```shell
$ history
```

Каждая строка пронумерована, а свежие команды — внизу списка. История выручает, когда нужно повторить недавнее действие, вспомнить, что именно вы делали, или восстановить ход лабораторной работы для отчёта.

## Переменные окружения Bash

Откуда shell знает, какая у вас оболочка, какой язык интерфейса, где искать программы? Эти настройки хранятся в переменных окружения — именованных значениях, которые доступны и самой оболочке, и любой запущенной из неё программе. По сути это общая «доска объявлений» для всех процессов.

Прочитать значение переменной можно через `echo`, поставив перед именем знак `$` — он говорит оболочке «подставь сюда значение». Так выглядит чтение переменной `SHELL`:

```shell
$ echo $SHELL
```

Чтобы увидеть сразу все переменные окружения, а не одну, есть команда `env`:

```shell
$ env
```

Завести собственную переменную просто — достаточно написать `имя=значение` без пробелов вокруг знака равенства:

```shell
$ OFFICE=caleston
```

Но у такой переменной есть ограничение: она живёт только в текущей оболочке и не передаётся программам, которые вы из неё запускаете (дочерним процессам). Чтобы переменную «увидели» и они, её нужно экспортировать командой `export`:

```shell
$ export OFFICE=caleston
```

Но и `export` решает задачу лишь наполовину: переменная исчезнет, как только вы закроете сессию. Чтобы она появлялась автоматически при каждом входе, её строку нужно записать в файл настройки оболочки в домашнем каталоге. В современных версиях Ubuntu (22.04 и новее) для этого служит `~/.profile` — он читается при входе пользователя в систему. Добавим в него нашу переменную (двойная стрелка `>>` дописывает строку в конец файла, не стирая остального):

```shell
$ echo "export OFFICE=caleston" >> ~/.profile
```

Теперь переменная будет появляться во всех новых сессиях. В уже открытой сессии файл не перечитывается сам — чтобы применить изменения здесь и сейчас, выполните `source ~/.profile`.

> Раньше для пользовательских переменных окружения также применяли файл `~/.pam_environment`, но начиная с Ubuntu 22.04 он по умолчанию не читается, поэтому использовать его не рекомендуется.

Не все переменные нужно создавать самому — многие система задаёт за вас. Например, в `LOGNAME` хранится имя пользователя, под которым вы вошли:

```shell
$ echo $LOGNAME
```

> **Типичные ошибки.**
>
> - **Пробелы вокруг `=` при присваивании.** `OFFICE = caleston` — это попытка запустить команду `OFFICE` с аргументами `=` и `caleston`. Bash присваивает только без пробелов: `OFFICE=caleston`.
> - **Переменная без `export` не «видна» дочерним процессам.** Если вы задали `OFFICE=caleston` и сразу запустили скрипт, обращающийся к `$OFFICE`, он получит пустую строку. Запускаемые программы наследуют только экспортированные переменные — пишите `export OFFICE=caleston`.
> - **Чтение переменной без `$`.** Команда `echo OFFICE` напечатает буквально слово `OFFICE`. Чтобы Bash подставил значение, нужно `echo $OFFICE`.
> - **Изменения в `~/.profile` не применяются «здесь и сейчас».** Файл читается при входе в систему; в уже открытой сессии его перечитывают вручную: `source ~/.profile`.

## Переменная `PATH`

Среди системных переменных есть одна особенно важная — `PATH`. Вспомните: внешняя команда — это файл где-то в файловой системе. Но вы же не пишете полный путь каждый раз, когда набираете `date` или `ls`. Откуда оболочка знает, где лежит нужный файл? Она ищет его в каталогах, перечисленных в `PATH`, — по очереди, пока не найдёт.

Посмотреть этот список каталогов можно так:

```shell
$ echo $PATH
```

Вы увидите несколько путей, разделённых двоеточиями, — это и есть «места, где Bash ищет программы». А чтобы узнать, в каком именно из них нашёлся файл конкретной команды, есть `which` (which — «который, какой»):

Синтаксис:

```shell
$ which <command>
```

Пример:

```shell
$ which obs-studio
```

Если `which` ничего не вывел, скорее всего, программа установлена в каталог, которого нет в `PATH`, — оболочка просто не знает, где её искать. Такой каталог можно добавить в `PATH`, и команда станет доступна по короткому имени:

```shell
$ export PATH=$PATH:/opt/obs/bin
$ which obs-studio
```

Обратите внимание на запись `$PATH:/opt/obs/bin` — мы не заменяем `PATH`, а дописываем новый каталог к старому значению. Как и любая переменная, это изменение живёт только в текущей сессии; чтобы оно сохранялось, его нужно добавить в профильный файл пользователя.

## Разобранный пример: добавляем личный каталог скриптов в `PATH`

**Задача.** У пользователя `michael` есть каталог `~/bin` с собственными скриптами. Нужно сделать так, чтобы запускать их можно было по короткому имени из любого каталога — и чтобы настройка сохранялась между сессиями.

**Решение.**

1. Убеждаемся, что каталог существует и в нём действительно есть исполняемый файл: `ls -l ~/bin`. Если файлов нет — создаём один для проверки: `echo 'echo hello from bin' > ~/bin/greet && chmod +x ~/bin/greet`.
2. Смотрим текущее значение `PATH`, чтобы случайно не стереть его: `echo $PATH`. Запоминаем, что оно содержит — туда мы дописываемся, а не заменяем целиком.
3. В текущей сессии расширяем `PATH`: `export PATH=$PATH:~/bin`. Запись `$PATH:` сохраняет старые каталоги; конструкция `:~/bin` добавляет новый в конец списка.
4. Проверяем, что Bash теперь находит скрипт: `which greet` должен показать `/home/michael/bin/greet`, а просто `greet` — выполнить его.
5. Чтобы настройка пережила новую сессию, дописываем строку в конец `~/.profile`: `echo 'export PATH=$PATH:~/bin' >> ~/.profile`. Кавычки одинарные — мы хотим записать выражение буквально, а не подставленным значением.
6. Применяем изменения в текущем терминале без перезахода: `source ~/.profile`.

**Проверка.** Откройте новый терминал и выполните `which greet` — путь к скрипту должен находиться без `export`. Команда `echo $PATH` покажет `~/bin` в конце списка.

## Настройка приглашения Bash

Та самая строка, которую вы видите перед курсором при входе в shell, — приглашение Bash — тоже не задана раз и навсегда. Её внешний вид настраивается, и это не косметика: грамотно настроенное приглашение постоянно держит перед глазами полезный контекст.

![bash-prompt](../images/ru_bash-prompt.svg)

Например, его можно настроить так, чтобы оно показывало имя пользователя и имя хоста — удобно, когда вы переключаетесь между несколькими серверами и легко перепутать, на каком находитесь. Приглашение вроде `[michael@prod-server]$` сразу говорит, кто вы и где, ещё до того, как наберёте первую команду.

За внешний вид приглашения отвечает специальная переменная окружения. Их несколько, но основная — `PS1`. Посмотреть текущее значение `PS1` можно той же командой `echo`:

![bash-prompt2](../images/ru_bash-prompt2.svg)

Посмотрим, что в ней сейчас:

```shell
$ echo $PS1
```

Изменить приглашение можно так же, как любую переменную. Для начала зададим самое простое — пусть оно показывает только слово `ubuntu-server`:

```shell
$ PS1="ubuntu-server"
$ echo $PS1
```

Постоянная строка — это, конечно, бедновато. Настоящая гибкость в том, что `PS1` понимает специальные последовательности-подстановки: каждая из них на лету заменяется на актуальное значение. Полный их набор — в таблице ниже.

| Код | Что выводит |
|---|---|
| `\d` | дата в формате «День Месяц Число», например `Tue May 26` |
| `\e` | управляющий символ ASCII (033) |
| `\h` | имя хоста (короткое) |
| `\H` | полное имя хоста (FQDN) |
| `\n` | перевод строки |
| `\r` | возврат каретки |
| `\s` | имя оболочки |
| `\t` | текущее время в 24-часовом формате `ЧЧ:ММ:СС` |
| `\T` | текущее время в 12-часовом формате `ЧЧ:ММ:СС` |
| `\@` | текущее время в 12-часовом формате с `am`/`pm` |
| `\A` | текущее время в 24-часовом формате `ЧЧ:ММ` |
| `\u` | имя текущего пользователя |
| `\w` | текущий рабочий каталог; `$HOME` сокращается до `~` |
| `\W` | базовое имя текущего рабочего каталога; `$HOME` сокращается до `~` |
| `\$` | `#`, если эффективный UID 0 (root), иначе `$` |

Собрав несколько таких последовательностей вместе, можно показывать дату, время, имя текущего пользователя, имя хоста и рабочий каталог сразу:

```shell
$ PS1="[\d \t \u@\h:\w ] $ "
```

Здесь `\d` — дата, `\t` — время, `\u` — имя пользователя, `\h` — имя хоста, `\w` — текущий каталог. Как и любое изменение переменной, оно действует только в текущей сессии; чтобы приглашение оставалось таким и дальше, строку `PS1=...` нужно добавить в `~/.profile`.

## Практическое задание

1. Выведите значение `SHELL`.
2. Создайте alias `ll` для команды `ls -l` и проверьте его.
3. Выведите последние команды через `history`.
4. Создайте переменную `COURSE=linux` и проверьте её через `echo $COURSE`.
5. Найдите расположение команды `date` через `which`.
6. Временно измените `PS1`, затем откройте новую сессию и проверьте, сохранилось ли изменение.
