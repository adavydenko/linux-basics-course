# Базовые команды Linux

- [Доп. видео](https://kodekloud.com/topic/basic-linux-commands-3/)

## Цели раздела

После раздела студент должен уметь:

- определять текущий рабочий каталог;
- просматривать содержимое каталогов;
- создавать файлы и каталоги;
- перемещаться по абсолютным и относительным путям;
- копировать, перемещать, переименовывать и удалять файлы;
- выбирать подходящую команду для просмотра содержимого файла.

Любая работа в терминале начинается одинаково: нужно понять, где вы находитесь, осмотреться и перейти туда, где предстоит работать. Команды из этого раздела вы будете набирать чаще всех остальных, поэтому освоим их сразу на практике — построим небольшую структуру каталогов своими руками и научимся в ней перемещаться.

Наша цель — создать структуру каталогов вида:

```text
/home/michael/        ← домашний каталог, уже существует
├── Asia/
│   ├── China/
│   └── India/
│       └── Mumbai/
├── Europe/
├── Africa/
└── America/
```

Самый верхний каталог, `/home/michael`, — это домашний каталог пользователя, он уже существует. Всё, что расположено внутри него, нам предстоит создать самостоятельно.

Первым делом стоит понять, где вы сейчас находитесь. Команда `pwd` (print working directory) выводит полный путь текущего каталога:

```shell
$ pwd
```

Чтобы увидеть, что лежит в текущем каталоге, используйте `ls` (list). Сразу после входа в систему она, скорее всего, ничего не покажет — каталог пуст:

```shell
$ ls
```

Теперь создадим первый каталог. За это отвечает `mkdir` (make directory):

```shell
$ mkdir Asia
```

`mkdir` принимает не одно имя, а сколько угодно — так несколько каталогов создаются за один раз:

```shell
$ mkdir Europe Africa America
```

Каталог создан, но мы по-прежнему находимся снаружи. Чтобы войти в него, используется `cd` (change directory):

```shell
$ cd Asia
```

Часто нужно создать сразу цепочку вложенных каталогов. Если просто выполнить `mkdir India/Mumbai`, команда завершится ошибкой: каталога `India` ещё нет. Флаг `-p` решает эту проблему — он создаёт по пути все недостающие каталоги:

```shell
$ mkdir -p India/Mumbai
```

Чтобы подняться на один уровень вверх, к родительскому каталогу, используйте `cd ..` — две точки всегда означают «каталог уровнем выше»:

```shell
$ cd ..
```

А команда `cd` без аргументов в любой момент возвращает вас в домашний каталог — удобно, когда вы потерялись в глубине файловой системы:

```shell
$ cd
```

### Абсолютные и относительные пути

![Absolute_and_relative_path_working_with_shell_I](../images/ru_Absolute_and_relative_path.svg)

Указать каталог команде можно двумя способами, и важно понимать разницу между ними.

**Абсолютный путь** отсчитывается от корня файловой системы `/` и полностью описывает расположение файла или каталога — например, `/home/michael/Asia`. Такой путь работает откуда угодно, потому что не зависит от того, где вы сейчас находитесь.

**Относительный путь** отсчитывается от текущего каталога — того самого, что показывает `pwd`. Если вы стоите в `/home/michael`, то путь `Asia` и абсолютный путь `/home/michael/Asia` ведут в одно и то же место.

Перейти в каталог по абсолютному пути:

```shell
$ cd /home/michael
```

Перейти в тот же каталог по относительному пути (если вы уже находитесь в `/home/michael`):

```shell
$ cd Asia
```

### Альтернатива `cd`: стек каталогов

![pushd_popd](../images/ru_pushd_popd.svg)

У `cd` есть пара полезных «родственников» — `pushd` и `popd`. Они не просто переходят в каталог, а запоминают, откуда вы пришли, складывая каталоги в стек. Это удобно, когда нужно ненадолго отлучиться в другой каталог и затем точно вернуться назад.

`pushd` переходит в указанный каталог и кладёт его в стек:

```shell
$ pushd /etc
```

Каждый следующий `pushd` добавляет в стек ещё один каталог:

```shell
$ pushd /var
$ pushd /tmp
$ pwd
/tmp
```

`popd` снимает каталог со стека и возвращает вас на шаг назад — в предыдущий запомненный каталог:

```shell
$ popd
```

### Работа с файлами и каталогами

Перейдём к командам, которые копируют, перемещают и удаляют файлы. Для примеров возьмём ту же структуру каталогов, но с добавленными файлами — задача в том, чтобы свести её к набору изменений ниже:

| Действие | Откуда | Куда / результат |
|---|---|---|
| Переместить | `Europe/Morocco/` | `Africa/Morocco/` |
| Переименовать | `Asia/India/Munbai/` | `Asia/India/Mumbai/` |
| Скопировать файл | `Asia/India/Mumbai/City.txt` | `Africa/Egypt/Cairo/City.txt` |
| Скопировать каталог | `Europe/UK/` | `Europe/UnitedKingdom/` |
| Удалить | `Europe/UK/London/Tottenham.txt` | — |

Команда `mv` (move) перемещает файл или каталог в другое место. И источник, и назначение можно указывать как абсолютным, так и относительным путём:

```shell
$ mv /home/michael/Europe/Morocco /home/michael/Africa/   # Абсолютный путь
$ mv Europe/Morocco Africa/                               # Относительный путь
```

Та же команда `mv` умеет и переименовывать: если «переместить» каталог в новое имя в том же расположении, он просто получит другое название:

```shell
$ mv Asia/India/Munbai Asia/India/Mumbai
```

Скопировать файл в другой каталог можно командой `cp` (copy) — исходный файл при этом остаётся на месте:

```shell
$ cp Asia/India/Mumbai/City.txt Africa/Egypt/Cairo
```

Удаляет файлы команда `rm` (remove). С ней стоит быть осторожным: корзины в терминале нет, и удалённый файл вернуть нельзя:

```shell
$ rm Europe/UK/London/Tottenham.txt
```

По умолчанию `cp` копирует только файлы. Чтобы скопировать каталог целиком, со всем его содержимым, добавьте флаг `-r` (recursive):

```shell
$ cp -r Europe/UK Europe/UnitedKingdom
```

Теперь — как заглянуть внутрь файла. Самый быстрый способ — `cat`: она выводит всё содержимое файла прямо в терминал:

```shell
$ cat Asia/India/Mumbai/City.txt
```

У `cat` есть и обратное применение. Если перенаправить её вывод в файл символом `>`, команда начнёт записывать в этот файл то, что вы вводите с клавиатуры:

```shell
$ cat > Africa/Egypt/Cairo/City.txt
  Cairo
  # Нажмите Ctrl + d для завершения ввода
```

Создать пустой файл проще всего командой `touch`:

```shell
$ touch /home/michael/Asia/China/Country.txt
```

`cat` показывает файл целиком, и для больших файлов это неудобно — текст просто пролетает мимо. Для постраничного просмотра есть `more`:

```shell
$ more new_file.txt
```

А ещё лучше — `less`: она тоже листает файл по страницам, но при этом позволяет прокручивать его в обе стороны и искать по тексту:

```shell
$ less new_file.txt
```

Наконец, вернёмся к `ls`. Сама по себе она показывает только имена, но с флагами становится гораздо полезнее. Флаг `-l` выводит подробный список — с правами доступа, владельцем, размером и датой изменения:

```shell
$ ls -l
```

Флаг `-a` добавляет к списку скрытые файлы (в Linux их имена начинаются с точки). Флаги можно объединять — `-la` покажет подробный список вместе со скрытыми файлами:

```shell
$ ls -la
```

Флаг `-t` сортирует список по времени изменения, новые файлы — сверху:

```shell
$ ls -lt
```

А `-r` разворачивает порядок сортировки. Сочетание `-ltr` — частый приём: подробный список от старых файлов к новым, так что самое свежее оказывается внизу, прямо перед приглашением:

```shell
$ ls -ltr
```

## Практическое задание

1. Создайте в домашнем каталоге структуру `course/linux/lab01`.
2. Перейдите в каталог `lab01` сначала по абсолютному пути, затем по относительному.
3. Создайте файл `notes.txt` через `touch`.
4. Запишите в файл одну строку через `cat > notes.txt`.
5. Скопируйте файл в `notes-copy.txt`.
6. Переименуйте `notes-copy.txt` в `notes-backup.txt`.
7. Просмотрите каталог через `ls -la`, `ls -lt` и `ls -ltr`.
8. Удалите резервную копию и проверьте результат.
