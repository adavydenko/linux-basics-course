# Лабораторная работа: оболочка и базовые команды

- [Доп. лабораторная](https://kodekloud.com/topic/lab-working-with-the-shell/)

## Цели лабораторной работы

После выполнения работы студент должен уметь:

- определять домашний каталог пользователя;
- различать команду и аргумент;
- определять тип команды;
- создавать, перемещать, переименовывать и удалять каталоги;
- использовать абсолютные пути в командах.

## Задания

Эта работа закрепляет навыки из первых разделов модуля: вы определите домашний каталог двумя разными способами, разберёте устройство команды и построите небольшую структуру каталогов, а затем измените её. Выполняйте задания по порядку — каждое следующее опирается на предыдущее.

1. Проверить домашний каталог пользователя `bob` через файл `/etc/passwd`. В этом файле система хранит сведения обо всех пользователях; нужное поле — шестое, поэтому строку с `bob` мы находим через `grep`, а затем вырезаем шестую колонку командой `cut`:

   ```shell
   $ grep bob /etc/passwd | cut -d ":" -f6
   ```

2. Проверить домашний каталог текущего пользователя — на этот раз проще, через встроенную переменную shell `HOME`:

   ```shell
   $ echo $HOME
   ```

3. В команде `echo Welcome` определить, чем является слово `Welcome`:

   ```shell
   $ echo Welcome
   ```

   `Welcome` — это аргумент команды `echo`.

4. Определить тип команды `git`:

   ```shell
   $ type git
   ```

5. Создать каталог `birds`. Здесь и далее путь задаётся абсолютным — от корня `/`, — чтобы команда работала независимо от того, где вы сейчас находитесь:

   ```shell
   $ mkdir /home/bob/birds
   ```

6. Создать сразу цепочку вложенных каталогов. Каталога `fish` ещё нет, поэтому без флага `-p` команда завершилась бы ошибкой — `-p` создаёт все недостающие каталоги по пути:

   ```shell
   $ mkdir -p /home/bob/fish/salmon
   ```

7. Создать ещё несколько каталогов, чтобы получить структуру для следующих шагов:

   ```shell
   $ mkdir -p /home/bob/mammals/elephant
   $ mkdir -p /home/bob/mammals/monkey
   $ mkdir /home/bob/birds/eagle
   $ mkdir -p /home/bob/reptile/snake
   $ mkdir -p /home/bob/reptile/frog
   $ mkdir -p /home/bob/amphibian/salamander
   ```

8. Переместить каталог `frog` из `reptile` в `amphibian` — лягушка попала не в ту группу, исправляем:

   ```shell
   $ mv /home/bob/reptile/frog /home/bob/amphibian
   ```

9. Переименовать каталог. Та же команда `mv` умеет и переименовывать: если «переместить» каталог в новое имя в том же расположении, он просто сменит название:

   ```shell
   $ mv /home/bob/reptile/snake /home/bob/reptile/crocodile
   ```

10. Удалить каталог `reptile` целиком. Флаг `-r` (recursive) нужен, чтобы удалить каталог вместе со всем содержимым; будьте внимательны — корзины в терминале нет, и удалённое не вернуть:

    ```shell
    $ rm -r /home/bob/reptile
    ```

## Что нужно сдать

Оформите отчёт по [единому формату](../00-Lab-Environment/01-Lab-Environment-Setup.ru.md#формат-отчёта-по-лабораторной-работе). Дополнительно для этой работы приложите:

- Список выполненных команд.
- Краткое объяснение команд `grep`, `cut`, `mkdir -p`, `mv`, `rm -r`.
- Вывод `ls` или `tree`, подтверждающий итоговую структуру каталогов.
