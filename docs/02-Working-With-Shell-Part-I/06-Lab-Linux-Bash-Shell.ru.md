# Лабораторная работа: Bash

- Перейти к [лабораторной работе](https://kodekloud.com/topic/lab-linux-bash-prompt/)

## Цели лабораторной работы

После выполнения работы студент должен уметь:

- проверять текущую оболочку пользователя;
- менять оболочку пользователя;
- читать и создавать переменные окружения;
- проверять переменную `PATH`;
- создавать постоянные alias;
- настраивать приглашение Bash.

## Задания

1. Проверить оболочку по умолчанию для текущего пользователя. Эта команда показывает оболочку, заданную пользователю, но не обязательно shell, который прямо сейчас выполняет команды:

   ```shell
   $ echo $SHELL
   ```

2. Изменить оболочку пользователя `bob` с Bash на Bourne Shell:

   ```shell
   $ chsh -s /bin/sh bob
   ```

3. Узнать значение переменной окружения `TERM`:

   ```shell
   $ echo $TERM
   ```

4. Создать переменную окружения `PROJECT=MERCURY` и сделать её постоянной, добавив в `~/.profile`:

   ```shell
   $ echo export PROJECT=MERCURY >> ~/.profile
   ```

5. Определить, какого каталога нет в переменной `PATH`:

   ```text
   /opt/caleston-code
   ```

6. Создать alias `up` для команды `uptime` и сделать его постоянным через `~/.profile`:

   ```shell
   $ echo alias up=uptime >> ~/.profile
   ```

7. Изменить приглашение пользователя `bob`, чтобы оно показывало дату в таком формате:

   ```text
   [Wed Apr 22]bob@caleston-lp10:~$
   ```

   Изменение должно быть постоянным.

   ```shell
   $ PS1='[\d]\u@\h:\w\$'
   ```

   Или добавить настройку в `~/.profile`:

   ```shell
   $ echo 'PS1=[\d]\u@\h:\w$' >> ~/.profile
   ```

## Что нужно сдать

- Вывод `echo $SHELL`, `echo $TERM`, `echo $PATH`.
- Команды, которыми были добавлены `PROJECT`, alias `up` и `PS1`.
- Краткое объяснение, почему изменения в `~/.profile` сохраняются между сессиями.

## Контрольные вопросы

1. Почему `echo $SHELL` не всегда показывает shell, который выполняет текущую команду?
2. Что означает переменная `TERM`?
3. Чем временное изменение `PS1` отличается от записи в `~/.profile`?
4. Почему изменение `PATH` может повлиять на запуск внешних команд?
