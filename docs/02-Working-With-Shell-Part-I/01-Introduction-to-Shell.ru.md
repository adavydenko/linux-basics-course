## Оболочка Linux

![Shell](../../images/Shell.PNG)

Оболочка Linux — это интерфейс командной строки (CLI), через который пользователь взаимодействует с операционной системой текстовыми командами. Такой интерфейс позволяет эффективно работать с ноутбуком, сервером или виртуальной машиной Linux.

Графический интерфейс удобен для многих повседневных действий, но на сервере он часто недоступен или избыточен. Именно поэтому командная строка, которую обычно называют **Linux shell**, остаётся основным инструментом администратора, разработчика и исследователя.

### Что такое shell

Shell — это программа, которая принимает команды пользователя, передаёт их операционной системе и возвращает результат в текстовом виде.

С помощью shell можно:

- перемещаться по файловой системе;
- создавать, копировать, переименовывать и удалять файлы;
- запускать программы;
- настраивать окружение;
- автоматизировать повторяемые действия.

Когда пользователь входит в shell, первым рабочим каталогом обычно становится его домашний каталог.

## Домашний каталог

![The-Home-Directory](../../images/The_Home_Directory.JPG)

У пользователя `michael` домашний каталог находится по пути `/home/michael`. Каталог `/home` создаётся системой и обычно содержит домашние каталоги обычных пользователей.

Имя домашнего каталога по умолчанию совпадает с именем пользователя: домашний каталог пользователя `michael` — `/home/michael`, а домашний каталог пользователя `allen` — `/home/allen`.

Домашний каталог нужен для хранения личных файлов пользователя. В нём пользователь может создавать, читать, изменять и удалять свои данные. Другие пользователи обычно не имеют доступа к этим файлам, если им явно не выданы соответствующие права.

Домашний каталог часто обозначается символом `~` (тильда). Например, `~/notes.txt` означает файл `notes.txt` в домашнем каталоге текущего пользователя.

## Приглашение командной строки

Приглашение командной строки (command prompt) — это строка, которую shell показывает перед вводом команды.

Его можно настроить так, чтобы оно показывало имя пользователя, имя хоста, текущий каталог, дату, время или другую полезную информацию. В примерах курса символ `~` в приглашении означает домашний каталог.

## Команды и аргументы

Чтобы взаимодействовать с Linux через shell, пользователь вводит команды.

Команда запускает программу или встроенную функцию shell для выполнения конкретной задачи. Например, команда `echo` выводит текст на экран:

```shell
$ echo
```

Аргумент — это входное значение для команды. Чтобы вывести слово `hello`, передадим его команде `echo` как аргумент:

```shell
$ echo hello
```

Некоторые команды могут работать без аргументов. Например, `uptime` показывает, сколько времени система работает с момента последней загрузки:

```shell
$ uptime
```

У команд также бывают опции. Опция изменяет поведение команды и часто называется флагом или переключателем. Например, флаг `-n` у `echo` выводит текст без завершающего перевода строки:

```shell
$ echo -n hello
```

## Типы команд

Команды в Linux можно разделить на две основные группы.

1. Встроенные команды (internal или built-in commands). Они являются частью самой оболочки. Примеры: `echo`, `cd`, `pwd`, `set`.
2. Внешние команды (external commands). Это отдельные исполняемые файлы или скрипты, которые находятся в файловой системе. Они могут поставляться вместе с дистрибутивом, устанавливаться пакетным менеджером или создаваться пользователем. Примеры: `mv`, `date`, `uptime`, `cp`.

Чтобы узнать тип команды, используйте `type`:

```shell
$ type cd
$ type date
```
