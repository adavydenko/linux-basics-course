# Справка в командной строке

- [Доп. видео](https://kodekloud.com/topic/command-line-help-3/)

Даже опытный администратор не помнит наизусть все опции всех команд — и это нормально. Куда полезнее уметь быстро найти нужную справку прямо в терминале, не открывая браузер. Linux несёт документацию в себе, и в этом разделе мы посмотрим, какими инструментами её доставать.

## Цели раздела

После раздела студент должен уметь:

- быстро узнавать назначение команды;
- открывать подробную справочную страницу;
- искать команды по ключевому слову;
- отличать краткую справку от полной документации.

## Зачем нужна справка

Команд в Linux сотни, у каждой — десятки опций, и держать всё это в памяти невозможно да и не нужно. Гораздо важнее выработать привычку: столкнулся с незнакомой командой — за несколько секунд проверил её назначение, синтаксис и нужные флаги, не прерывая работу. Справочные инструменты ниже расположены по принципу «от короткого ответа к подробному»: сначала те, что отвечают одной строкой, потом — полная документация.

## `whatis`

Самый быстрый вопрос, который возникает при встрече с незнакомой командой: «а что она вообще делает?». На него отвечает `whatis` — она печатает краткое описание команды ровно в одну строку.

Синтаксис:

```shell
$ whatis <command>
```

Пример:

```shell
$ whatis date
```

В ответ вы увидите одну строку вроде «date — print or set the system date and time». Этого достаточно, чтобы вспомнить назначение утилиты, но недостаточно, чтобы узнать её опции — для этого нужен инструмент поподробнее.

## `man`

Когда одной строки мало, на помощь приходит `man` (manual — руководство). Почти каждая команда поставляется со справочной страницей — `man page`, — где собрано подробное описание: синтаксис, полный список опций и нередко примеры использования. Это основной справочник Linux, и обращаться к нему стоит без стеснения.

Синтаксис:

```shell
$ man <command>
```

Пример:

```shell
$ man date
```

`man` открывает страницу в постраничном просмотрщике, поэтому текст не пролетает мимо — его можно листать. Полезно сразу запомнить несколько клавиш управления:

- `Space` — перейти на страницу ниже;
- `b` — вернуться на страницу выше;
- `/text` — найти на странице слово `text`;
- `q` — выйти из просмотра.

Если выйти не нажатием `q`, а закрыть терминал, можно растеряться — поэтому `q` стоит запомнить в первую очередь.

## `-h` и `--help`

Часто полная `man`-страница избыточна — нужно просто вспомнить, какие у команды есть опции. На этот случай многие команды поддерживают краткую встроенную справку через флаг `-h` или его длинную форму `--help`:

```shell
$ date -h
$ date --help
```

Вывод помещается в пару экранов и показывает основные параметры — это быстрее, чем открывать `man`. Но единого стандарта здесь нет: одни команды понимают только `--help`, другие — только `-h`, а у некоторых нет ни того ни другого. Если флаг не сработал, не пугайтесь — переходите к `man`.

## `apropos`

Все предыдущие команды отвечают на вопрос «что делает вот эта команда?». Но бывает наоборот: вы знаете, *что* хотите сделать, но не помните, *как* называется нужная утилита. Здесь выручает `apropos` — она ищет по названиям и описаниям всех справочных страниц и возвращает список команд, в которых встретилось ваше ключевое слово.

Синтаксис:

```shell
$ apropos <keyword>
```

Пример:

```shell
$ apropos modpr
```

В ответ вы получите список подходящих команд с их короткими описаниями — по сути это `whatis` наоборот: не «команда → описание», а «слово → команды». Если поиск ничего не нашёл, скорее всего, дело в опечатке или слишком узком слове — проверьте написание или попробуйте более общий термин.

## Практическое задание

1. Узнайте назначение команд `date`, `passwd`, `uptime` через `whatis`.
2. Откройте `man date` и найдите внутри страницы слово `format`.
3. Сравните вывод `date --help` и `man date`.
4. Найдите команды, связанные с password, через `apropos password`.
