# Диагностика Linux

Диагностика — это умение быстро понять, что именно сломалось: права, диск, память, сеть, сервис, пакет или сама команда. Этот раздел собирает базовый порядок действий.

## Цели раздела

После раздела студент должен уметь:

- проверять симптомы проблемы;
- читать логи;
- диагностировать место на диске и память;
- проверять сервисы systemd;
- проверять сеть;
- оформлять краткий диагностический отчёт.

## Общий подход

1. Сформулировать симптом.
2. Воспроизвести проблему.
3. Посмотреть точный текст ошибки.
4. Проверить самые вероятные причины.
5. Зафиксировать команды и вывод.
6. Сделать минимальное изменение и проверить снова.

## Логи

Логи systemd:

```shell
$ journalctl
$ journalctl -u ssh
$ journalctl -u nginx --since "1 hour ago"
```

Сообщения ядра:

```shell
$ dmesg
$ dmesg | tail
```

## Диск

Свободное место:

```shell
$ df -h
```

Размер каталогов:

```shell
$ du -sh *
$ du -sh /var/log
```

Найти большие файлы:

```shell
$ find . -type f -size +100M
```

## Память и процессы

```shell
$ free -h
$ uptime
$ top
$ ps aux --sort=-%mem | head
```

## Сервисы

```shell
$ systemctl status nginx
$ systemctl restart nginx
$ journalctl -u nginx
```

Если сервис не стартует, сначала смотрите `systemctl status`, затем `journalctl -u`.

## Сеть

```shell
$ ip addr
$ ip route
$ ping 8.8.8.8
$ curl -v http://localhost:8080
$ ss -tuln
```

Если IP-адрес пингуется, а доменное имя нет, вероятна проблема с DNS.

## Права доступа

```shell
$ ls -l file.txt
$ id
$ groups
```

Проблемы прав часто проявляются как `Permission denied`.

## Практическое задание

Подготовьте диагностический отчёт для одной искусственной проблемы:

1. Не хватает места на диске.
2. Сервис не запущен.
3. Порт не слушается.
4. Нет прав на файл.
5. Команда не найдена.

В отчёте должны быть: симптом, команды проверки, вывод, вывод о причине и исправление.
