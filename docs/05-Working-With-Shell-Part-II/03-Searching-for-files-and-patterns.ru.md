# Поиск файлов и шаблонов

- [Доп. видео](https://kodekloud.com/topic/searching-for-files-and-patterns/)

«Где-то у меня был этот файл» и «в каком из логов мелькала эта ошибка» — два вопроса, которые возникают у любого, кто работает в Linux. Отвечают на них три инструмента, и важно понимать, чем они отличаются: `locate` и `find` ищут сами файлы, а `grep` — строки текста внутри них. Разберём все три и заодно увидим, почему у быстрого `locate` есть подвох.

## Цели раздела

После раздела студент должен уметь:

- искать файл через `locate`;
- понимать ограничение базы `locate`;
- искать файлы через `find`;
- искать строки внутри файлов через `grep`;
- использовать основные флаги `grep`: `-i`, `-r`, `-v`, `-w`, `-A`, `-B`.

## `locate`

Начнём с самого быстрого способа. Нужно найти все файлы с именем `City.txt` — команда `locate` выдаст результат почти мгновенно:

```shell
$ locate City.txt
```

В ответ вы получите список путей, совпадающих с шаблоном. Скорость у `locate` берётся не из воздуха: команда ищет не по файловой системе напрямую, а по заранее построенной базе `mlocate.db`. В этом и её сила, и её слабость.

Слабость в том, что база обновляется не сразу. Если система установлена недавно или файл создан только что, `locate` может его «не увидеть» — в базе его попросту ещё нет. Лекарство — обновить базу вручную:

```shell
$ sudo updatedb
```

Обратите внимание: `updatedb` требует прав `root`, поэтому запускается через `sudo`.

## `find`

Когда нужна не скорость, а актуальность, на помощь приходит `find`. В отличие от `locate`, он ищет прямо в файловой системе, поэтому всегда видит самое свежее состояние — но и работает медленнее. Команде сначала указывают каталог, в котором искать, а затем условие поиска.

Например, найти файл `City.txt` внутри `/home/michael`:

```shell
$ find /home/michael -name City.txt
```

Условие `-name` ищет по имени; у `find` есть и другие условия, но для начала достаточно этого.

## `grep`

Третий инструмент решает другую задачу — он ищет не файлы, а строки текста внутри них. По умолчанию `grep` (global regular expression print) чувствителен к регистру: `Second` и `second` для него разные слова.

Найдём строки со словом `second` в файле `sample.txt`:

```shell
$ grep second sample.txt
```

В ответ `grep` выведет все строки файла, где встретилось совпадение.

Чтобы регистр не учитывался, добавьте флаг `-i` (ignore case):

```shell
$ grep -i capital sample.txt
```

Флаг `-r` (recursive) распространяет поиск на весь каталог — `grep` пройдёт по всем файлам внутри:

```shell
$ grep -r "third Line" /home/michael
```

А `-v` (invert) переворачивает логику: вместо совпавших строк `grep` покажет все, которые шаблону не соответствуют:

```shell
$ grep -v "printed" sample.txt
```

## Поиск целого слова

По умолчанию `grep` находит шаблон и внутри слов — поиск `exam` совпадёт и с `example`, и с `exams`. Флаг `-w` (word) ограничивает поиск целыми словами:

```shell
$ grep -w exam examples.txt
```

Флаги свободно комбинируются. Скажем, `-vw` выведет все строки, в которых нет целого слова `exam`:

```shell
$ grep -vw exam examples.txt
```

Иногда мало найти строку — хочется увидеть и её окружение. Флаг `-A` (after) добавляет к совпадению указанное число строк после него:

```shell
$ grep -A1 Arsenal premier-league-table.txt
```

А `-B` (before) — столько же строк перед совпадением:

```shell
$ grep -B1 4 premier-league-table.txt
```

`-A` и `-B` можно применять вместе, чтобы увидеть совпадение в контексте сразу с обеих сторон:

```shell
$ grep -A1 -B1 Chelsea premier-league-table.txt
```

## Практическое задание

1. Найдите файл по имени через `find`.
2. Обновите базу `locate` и найдите тот же файл через `locate`.
3. Найдите строки с `error` без учёта регистра.
4. Найдите строки, где нет слова `debug`.
5. Найдите совпадение и выведите по одной строке до и после него.
