# Лабораторная работа: Vi

- [Доп. лабораторная](https://kodekloud.com/topic/lab-vi-editor-3/)

## Цели лабораторной работы

После выполнения работы студент должен уметь:

- входить в режим вставки;
- возвращаться в командный режим;
- удалять символы и строки;
- сохранять файл;
- искать текст внутри файла;
- отменять изменения.

## Задания

Эта работа закрепляет главное в `vi` — умение свободно переключаться между режимами и выполнять в каждом нужные действия. Задания проведут вас по полному циклу редактирования: войти в режим вставки, что-то изменить, удалить, найти, отменить и сохранить результат.

1. Перейти в режим вставки:

   ```text
   Нажмите i
   ```

2. Выйти из режима вставки и вернуться в командный режим:

   ```text
   Нажмите Esc
   ```

3. Удалить символ:

   ```text
   Переместите курсор на нужный символ и нажмите x.
   ```

4. Изменить содержимое файла на `Welcome to KodeKloud` и сохранить файл:

   ```text
   Нажмите i, удалите старое содержимое и введите новый текст.
   Затем нажмите Esc и выполните :w!
   ```

5. В конфигурации Apache изменить порт с `80` на `5000`:

   ```text
   Перейдите в режим вставки и замените 80 на 5000 в нужной строке.
   ```

6. Удалить строку, которая начинается с `LogFormat`:

   ```text
   Перейдите на нужную строку и нажмите dd.
   ```

7. Отменить предыдущее изменение:

   ```text
   Нажмите u.
   ```

8. Найти символ `6` в файле:

   ```text
   В командном режиме выполните /6
   ```

## Возврат к Бобу

> **Возврат к Бобу.** Боб готовит сервер: ему нужно установить пакет `nginx`, поправить одну строку в его конфигурации `/etc/nginx/nginx.conf`, а затем собрать архив с логами `/var/log/nginx/` за последний день и отправить коллеге. На рабочей машине только `vi` — других редакторов нет. Опишите план: какие команды Боб выполнит по порядку — от установки до получения архива? Где он будет переключаться между режимами `vi`, какие потоки перенаправит, чтобы лишние сообщения не попали в архив, и как одной строкой убедится, что архив действительно содержит только нужные файлы?
