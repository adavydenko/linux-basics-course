# Перенаправление ввода-вывода

- Перейти к [видеоуроку](https://kodekloud.com/topic/io-redirection/)

В этом разделе мы разберём перенаправление ввода-вывода и стандартные потоки Linux.

## Цели раздела

После раздела студент должен уметь:

- объяснить, что такое `STDIN`, `STDOUT`, `STDERR`;
- перенаправлять обычный вывод в файл;
- перенаправлять ошибки в файл;
- добавлять вывод в конец файла;
- использовать `/dev/null`;
- соединять команды через pipe;
- использовать `tee`.

## Стандартные потоки

Когда запускается команда, Linux создаёт три стандартных потока:

- `STDIN` — стандартный ввод;
- `STDOUT` — стандартный вывод;
- `STDERR` — стандартный поток ошибок.

![io](../../images/io.PNG)

Через перенаправление можно отправить `STDIN`, `STDOUT` и `STDERR` в файл или другой поток.

## Перенаправление `STDOUT`

Записать обычный вывод в файл:

```shell
$ echo $SHELL > shell.txt
```

Добавить обычный вывод в конец существующего файла:

```shell
$ echo $SHELL >> shell.txt
```

## Перенаправление `STDERR`

Чтобы перенаправить только ошибки, используется дескриптор `2`:

```shell
$ cat missing_file 2> error.txt
```

Добавить ошибки в конец файла:

```shell
$ cat missing_file 2>> error.txt
```

Если ошибки не нужно показывать и сохранять, их можно отправить в `/dev/null`:

```shell
$ cat missing_file 2> /dev/null
```

## Pipe

Pipe соединяет несколько команд. Вывод первой команды становится вводом второй. Pipe обозначается символом `|`.

```shell
$ grep Hello sample.txt | less
```

![pipe](../../images/pipe.PNG)

## `tee`

`tee` читает данные из `STDIN` и одновременно пишет их в файл и в `STDOUT`.

```shell
$ echo $SHELL | tee shell.txt
```

Чтобы добавить в конец файла, используйте `tee -a`:

```shell
$ echo "This is the bash shell" | tee -a shell.txt
```

![tee](../../images/tee.PNG)

## Практическое задание

1. Запишите вывод `echo $SHELL` в файл.
2. Добавьте ещё одну строку в тот же файл через `>>`.
3. Выполните команду с ошибкой и перенаправьте `STDERR` в отдельный файл.
4. Соедините `grep` и `less` через pipe.
5. Запишите вывод команды одновременно на экран и в файл через `tee`.
