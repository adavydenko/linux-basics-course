# Архивация и сжатие файлов

- [Доп. видео](https://kodekloud.com/topic/file-compression-and-archival/)

Нужно переслать коллеге целый каталог проекта или освободить место на диске под завязку забитого сервера — и тут выясняется, что десятки разрозненных файлов неудобно ни передавать, ни хранить. В Linux эту задачу решают два разных, хотя и часто работающих вместе действия: архивирование (собрать много файлов в один) и сжатие (уменьшить объём данных). Важно с самого начала их не путать — это поможет понять, почему `tar` и `gzip` существуют по отдельности и зачем их обычно применяют вместе.

## Цели раздела

После раздела студент должен уметь:

- смотреть размер файла или каталога;
- создавать архив `tar`;
- просматривать содержимое архива;
- распаковывать архив;
- сжимать и распаковывать файлы через `gzip`, `bzip2`, `xz`;
- читать сжатые текстовые файлы без предварительной распаковки.

## Просмотр размера файлов

Прежде чем что-то архивировать или сжимать, полезно понимать, с каким объёмом вы имеете дело. За это отвечает команда `du` (disk usage) — она показывает, сколько места занимает файл или каталог.

По умолчанию `du` выдаёт размер в килобайтах:

```shell
$ du -sk test.img
```

Флаг `-h` (human-readable) переводит число в привычные килобайты, мегабайты и гигабайты — читать такой вывод гораздо удобнее:

```shell
$ du -sh test.img
```

Размер одного файла можно узнать и без `du` — например, из подробного списка `ls -lh`, где он показан в одной из колонок:

```shell
$ ls -lh test.img
```

## Архивация файлов

Первое из двух действий — архивирование. Команда `tar` берёт несколько файлов и каталогов и склеивает их в один файл. Сам по себе `tar` ничего не сжимает: его задача — упаковка, а не уменьшение объёма. Название исторически означает «tape archive» (архив для магнитной ленты), а получившиеся файлы по традиции называют tarball.

Создадим архив из трёх файлов и сразу посмотрим на результат:

```shell
$ tar -cf test.tar file1 file2 file3
$ ls -ltr test.tar
```

Здесь две опции:

- `-c` (create) — создать архив;
- `-f` (file) — указать имя архивного файла; имя идёт сразу после `-f`.

Чтобы заглянуть внутрь архива, не распаковывая его, замените `-c` на `-t` (list):

```shell
$ tar -tf test.tar
```

А распаковывает архив опция `-x` (extract):

```shell
$ tar -xf test.tar
```

Вот тут архивирование и сжатие встречаются. Опция `-z` говорит `tar`: после упаковки сразу прогони результат через gzip. Так одной командой получается сжатый архив `test.tar.gz`:

```shell
$ tar -zcf test.tar.gz file1 file2 file3
```

## Сжатие

Теперь о втором действии — собственно сжатии, которое уменьшает объём данных. В Linux обычно встречаются три инструмента, и различить их проще всего по расширению файла:

- `bzip2` — расширение `.bz2`;
- `gzip` — расширение `.gz`;
- `xz` — расширение `.xz`.

```shell
$ bzip2 test.img
$ gzip test1.img
$ xz test2.img
```

Обратите внимание: каждая из этих команд заменяет исходный файл сжатым, дописывая к имени своё расширение. Насколько сильно «похудеет» файл, заранее не угадать — это зависит от типа данных, выбранного алгоритма и уровня сжатия.

Распаковка — зеркальные команды, по одной на каждый алгоритм:

```shell
$ bunzip2 test.img.bz2
$ gunzip test1.img.gz
$ unxz test2.img.xz
```

## Чтение сжатых файлов

Часто сжатый текстовый файл нужно просто прочитать — и распаковывать его на диск ради этого расточительно. Для таких случаев есть «прозрачные» аналоги `cat`: `zcat`, `bzcat` и `xzcat` распаковывают данные на лету и сразу выводят содержимое, не оставляя распакованной копии на диске:

```shell
$ bzcat hostfile.txt.bz2
$ zcat hostfile.txt.gz
$ xzcat hostfile.txt.xz
```

Имя команды подсказывает алгоритм: `zcat` — для `.gz`, `bzcat` — для `.bz2`, `xzcat` — для `.xz`.

## Практическое задание

1. Создайте несколько текстовых файлов.
2. Соберите их в архив `test.tar`.
3. Посмотрите содержимое архива через `tar -tf`.
4. Распакуйте архив в отдельный каталог.
5. Создайте архив `test.tar.gz` через `tar -zcf`.
6. Сравните размеры через `ls -lh` и `du -sh`.
