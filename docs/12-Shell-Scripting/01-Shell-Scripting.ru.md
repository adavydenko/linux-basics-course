# Shell scripting

Shell-скрипт — это файл с командами shell, который можно запускать как программу. Скрипты нужны для автоматизации повторяемых действий: подготовки каталогов, обработки файлов, запуска расчётов, проверки логов.

## Цели раздела

После раздела студент должен уметь:

- создавать простой shell-скрипт;
- передавать аргументы в скрипт;
- использовать переменные, условия и циклы;
- проверять код завершения команды;
- писать скрипт, который можно повторно запускать и читать через неделю.

## Первый скрипт

Создайте файл `hello.sh`:

```shell
#!/usr/bin/env bash

echo "Hello, Linux"
```

Сделайте файл исполняемым и запустите:

```shell
$ chmod +x hello.sh
$ ./hello.sh
```

Первая строка называется shebang. Она указывает, какой интерпретатор должен выполнить файл.

## Переменные

```shell
#!/usr/bin/env bash

course="linux"
echo "Course: $course"
```

При присваивании вокруг `=` не ставятся пробелы.

## Аргументы командной строки

```shell
#!/usr/bin/env bash

echo "Script name: $0"
echo "First argument: $1"
echo "Second argument: $2"
echo "All arguments: $@"
```

Запуск:

```shell
$ ./args.sh input.csv output.txt
```

## Условия

```shell
#!/usr/bin/env bash

file="$1"

if [ -f "$file" ]; then
  echo "File exists: $file"
else
  echo "File not found: $file"
  exit 1
fi
```

Кавычки вокруг переменных важны: они защищают от пробелов и пустых значений.

## Циклы

Цикл по файлам:

```shell
for file in *.log; do
  echo "$file"
  wc -l "$file"
done
```

Цикл по строкам файла:

```shell
while IFS= read -r line; do
  echo "$line"
done < input.txt
```

## Код завершения

Каждая команда завершается с кодом. `0` обычно означает успех, ненулевое значение — ошибку.

```shell
$ grep "ERROR" app.log
$ echo $?
```

В скриптах полезно включать строгий режим:

```shell
set -euo pipefail
```

Он помогает раньше замечать ошибки, но требует аккуратного обращения с командами, которые могут законно вернуть ненулевой код.

## Практическое задание

Напишите скрипт `log-summary.sh`, который:

1. Принимает путь к лог-файлу первым аргументом.
2. Проверяет, что файл существует.
3. Печатает количество строк.
4. Печатает количество строк с `ERROR`.
5. Печатает последние 5 строк файла.
6. Завершается с кодом `1`, если файл не найден.
